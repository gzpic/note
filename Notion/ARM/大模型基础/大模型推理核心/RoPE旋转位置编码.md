# RoPE 旋转位置编码

## 1. RoPE 是什么

RoPE（Rotary Position Embedding，旋转位置编码）是大模型里很常用的一种位置编码方法。

典型模型包括：

- LLaMA
- Qwen
- GPT-NeoX

它的核心做法不是把位置向量直接加到 embedding 上，而是：

> 对 `Q` 和 `K` 的每对维度做旋转，把位置信息编码进 attention 计算里。

## 2. 为什么需要位置编码

Transformer 本身对顺序不敏感。

如果没有位置编码，模型只看见 token 集合，而不知道：

- 谁在前
- 谁在后
- 两个 token 相隔多远

所以必须额外给模型提供位置信息。

## 3. RoPE 的核心思想

RoPE 不再做：

- `token_embedding + position_embedding`

而是改成：

- 对 `Q` 做旋转
- 对 `K` 做旋转
- 再用旋转后的 `Q / K` 做 attention

也就是：

```text
Q → Q_rot
K → K_rot
Attention = Q_rot · K_rot
```

## 4. 它到底是绝对位置还是相对位置

这个问题很容易混淆，最准确的说法是：

> RoPE 在实现上是绝对位置编码，但在 attention 效果上体现为相对位置。

原因是：

- 每个 token 的旋转角度由它自己的位置 `pos` 决定
- 但在 `Q_i · K_j` 的结果里，最终只和 `j - i` 的相对距离相关

所以可以记成：

- 形式上：绝对位置
- 结果上：相对位置

## 5. 最标准的完整公式

设：

- 位置为 `p`
- 维度为 `d`
- 第 `i` 个二维维度对的频率为：

```text
theta_i = 1 / 10000^(2i / d)
```

则对应位置上的旋转角度为：

```text
phi_{p,i} = p * theta_i
```

把向量拆成二维对：

```text
(x_{2i}, x_{2i+1})
```

旋转后得到：

```text
x'_{2i}   = x_{2i} * cos(phi_{p,i}) - x_{2i+1} * sin(phi_{p,i})
x'_{2i+1} = x_{2i} * sin(phi_{p,i}) + x_{2i+1} * cos(phi_{p,i})
```

这就是 RoPE 最核心的公式。

## 6. 为什么这样会产生相对位置效果

RoPE 作用在 `Q_i` 和 `K_j` 后，attention 中的内积可以推成：

```text
(Q'_i)^T K'_j = Q_i^T R(phi_j - phi_i) K_j
```

关键在于：

```text
phi_j - phi_i ∝ j - i
```

所以 attention 的位置依赖最终只和相对距离相关。

这就是为什么 RoPE 特别适合 Transformer。

## 7. 一个简单例子

要注意一点：

> RoPE 实际上要求偶数维，因为它是每两维一组做二维旋转。

如果拿一个简单的 3 维向量举例，可以理解成：

- 前两维旋转
- 最后一维不动

例如：

```text
q = [1, 2, 3]
```

设：

- `pos = 1`
- `theta = 30°`
- `cos(theta) = 0.866`
- `sin(theta) = 0.5`

则前两维旋转后：

```text
x' = 1 * 0.866 - 2 * 0.5   = -0.134
y' = 1 * 0.5   + 2 * 0.866 =  2.232
```

所以结果变成：

```text
q_rot = [-0.134, 2.232, 3]
```

真实模型里当然不是 3 维，而是例如 `head_dim = 128`，也就是要对 64 对二维向量分别做旋转。

## 8. RoPE 不是“相邻 token 向量之间的角度”

这是一个很容易误解的点。

RoPE 旋转的对象不是：

- 两个 token 之间的向量夹角

而是：

- 同一个 token 向量内部的两个相邻维度

也就是说，RoPE 的操作单位是：

```text
(x0, x1), (x2, x3), (x4, x5), ...
```

每一对维度按当前位置 `pos` 旋转一个角度。

## 9. 每个角度是怎么计算的

角度本质上就是：

> 位置 `pos` × 一个和维度对有关的频率

也就是：

```text
phi_{p,i} = p / 10000^(2i / d)
```

越靠前的维度对，频率越高，旋转得更快；
越靠后的维度对，频率越低，旋转得更慢。

这样模型可以同时表达：

- 短距离关系
- 长距离关系

## 10. 在推理流程里的位置

RoPE 通常出现在：

```text
Embedding
  ↓
QKV Linear
  ↓
RoPE(Q, K)
  ↓
QK^T
  ↓
Softmax
  ↓
V
```

也就是：

- 先算出 `Q / K / V`
- 再对 `Q` 和 `K` 做 RoPE
- 然后进入 attention

## 11. 为什么它在推理框架里很重要

在推理框架实现里，RoPE 通常是一个很热点的小算子。

原因是：

- 每层都要做
- 每个 token 都要做
- 常常紧挨着 QKV 输出

所以工程上经常会考虑：

- 把 `QKV + RoPE + reshape` 融合
- 用 `half2 / float2` 做二维旋转
- 减少单独 kernel launch

这在 Jetson 这类端侧设备上尤其有价值。

## 12. RoPE 的优点

RoPE 的主要优点有：

- 天然带相对位置效果
- 不引入可训练参数
- 长上下文扩展性较好
- 和很多 LLM 架构兼容性很好

## 13. 一句话总结

RoPE 的本质是：

> 把向量每两维看成一个二维平面，按 token 的绝对位置做旋转，从而让 attention 结果自然体现相对位置关系。
