# 对称与非对称量化

## 1. 先看直觉

对称量化：

- 以 0 为中心
- 只有 `scale`
- `zero_point = 0`

非对称量化：

- 不一定以 0 为中心
- 同时有 `scale` 和 `zero_point`
- 能更贴近偏移分布

## 2. 数学表达

### 2.1 对称量化

```text
q = round(x / s)
x ≈ q * s
zero_point = 0
```

### 2.2 非对称量化

```text
q = round(x / s + z)
x ≈ (q - z) * s
zero_point = z
```

## 3. 工程差异

| 维度 | 对称量化 | 非对称量化 |
| --- | --- | --- |
| 参数 | `scale` | `scale + zero_point` |
| kernel 复杂度 | 低 | 高 |
| GEMM 融合 | 好 | 差 |
| 硬件友好度 | 高 | 低 |
| 推理性能 | 更好 | 更差 |
| 分布贴合度 | 一般 | 更好 |

## 4. 为什么 LLM 推理更偏对称量化

核心原因是：

- zero_point 会让 kernel 更复杂
- GEMM 和 Attention 更难融合
- 高性能推理更依赖对称、规则的数值路径

所以在大模型推理中，尤其是参与 Attention 的张量，往往更偏好对称量化。

## 5. 非对称量化什么时候有意义

非对称量化在这些场景更有价值：

- 某些激活分布整体偏移明显
- 离线校准时希望更贴近数据范围
- 不把性能放在第一位

## 6. 工程上的常见结论

- 推理 runtime 更偏对称量化
- 校准和分析阶段可以更灵活地利用非对称统计
- 权重和高性能 GEMM 路线通常更偏对称量化

## 7. 面试速记版

- 对称量化只有 scale，zero_point 为 0
- 非对称量化多一个 zero_point，表达能力更强
- LLM 推理里更常用对称量化，因为它更利于 GEMM 和 Attention 的高性能实现
