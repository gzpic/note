# ONNX计算图优化分类与效能排序

## 1. 先说结论

如果从工程里的“平均收益”来看，基于 ONNX 计算图的优化通常可以按下面这个顺序理解：

1. 量化 / 混合精度重写
2. 大子图下沉到专用后端
3. 算子融合 / 子图模式融合
4. 布局优化 / 数据排布优化
5. 常量折叠
6. 冗余节点消除 / 图简化

这个顺序不是绝对真理，但很适合当部署时的默认优先级。

## 2. 第一类：量化 / 混合精度重写

这类优化通常收益最大。

它做的事情包括：

- 把 `FP32` 改成 `FP16 / BF16 / INT8`
- 在图中插入或消化 `Quantize / Dequantize`
- 把部分算子改写成低精度执行路径

为什么收益大：

- 直接降低权重带宽
- 直接降低激活带宽
- 在支持低精度硬件时还能切到更快 kernel

风险：

- 依赖硬件是否真的高效支持
- 依赖模型是否适合该精度
- 可能带来精度下降

## 3. 第二类：大子图下沉到专用后端

这类优化很多时候和量化属于第一梯队。

本质上是：

- 把一整段 ONNX 子图交给更强的执行后端
- 例如 TensorRT、OpenVINO、MIGraphX 之类

为什么收益大：

- 不只是优化几个节点
- 而是整段子图切到更高性能的执行器
- 后端还能继续做 kernel 选择、autotune、fusion

一句话：

> 这是“换执行器”，不是只做图简化。

## 4. 第三类：算子融合 / 子图模式融合

这是最典型的纯计算图优化。

常见例子：

- `MatMul + Add`
- `Conv + BN + Activation`
- `LayerNorm Fusion`
- `GELU Fusion`
- `Attention Fusion`

为什么收益高：

- 减少 kernel launch
- 减少中间 tensor 落地和回读
- 更容易命中 fused kernel

对 Transformer 和 CNN 都非常重要。

## 5. 第四类：布局优化 / 数据排布优化

这类优化本质上是在不改数学语义的前提下，调整 tensor 的 layout，让后端更容易跑高效 kernel。

它主要改善的是：

- 数据访问模式
- cache 友好度
- 硬件适配度

为什么通常排在融合后面：

- 它的收益更依赖模型和平台
- 不是所有模型都对 layout 同样敏感

可以把它理解成：

- 融合是在“减少步骤”
- 布局优化是在“让数据更适合硬件”

## 6. 第五类：常量折叠

常量折叠是很稳的基础优化，但平均收益通常没前面几类大。

它做的事情包括：

- 编译期先把常量子图算掉
- 把固定表达式直接折叠成常量
- 减少运行时无意义计算

价值主要是：

- 缩短执行路径
- 给后续优化创造条件

## 7. 第六类：冗余节点消除 / 图简化

这类优化通常收益最稳定，但单独看收益也最小。

常见内容：

- 删除 `Identity`
- 删除推理时无意义的 `Dropout`
- 去掉冗余 `Cast / Reshape / Transpose`
- 消除无用分支和无用输出

它更像是在：

- 清垃圾
- 减少图噪音
- 给融合和后端下沉铺路

## 8. 为什么这个排序不是绝对的

因为不同模型关注点不同。

### 8.1 Transformer

更常见的实际顺序是：

- 量化 / 混合精度
- 专用后端下沉
- Attention / LayerNorm / GELU 融合
- 其他图优化

### 8.2 CNN

更常见的顺序是：

- 量化 / 混合精度
- 专用后端下沉
- Conv 类融合
- layout 优化
- 基础图简化

### 8.3 小模型或已经很干净的图

可能会表现成：

- 量化
- EP 下沉
- 其余优化收益都比较有限

## 9. 如果只按 ONNX Runtime 的图优化层级去看

可以粗略分成：

- Basic
- Extended
- Layout

其中：

- `Basic` 更偏常量折叠和简化
- `Extended` 更偏融合
- `Layout` 更偏数据排布优化

如果只在这三类里比较，通常会认为：

- `Extended / Layout` 往往比 `Basic` 更有性能收益

## 10. 工程上的最实用顺序

如果你真的在做 ONNX 部署，比较划算的优化顺序通常是：

1. 先看能不能下沉到高性能 EP
2. 再看能不能做 `FP16 / INT8`
3. 再检查是否命中核心 fusion
4. 最后再看 layout 和基础图简化

因为前两步最容易拉开差距，后面的优化更像继续抠细节。

## 11. 面试速记版

- ONNX 图优化里，平均收益最大的通常是量化和专用后端下沉
- 纯图优化里最重要的是算子融合
- layout 优化依赖平台
- 常量折叠和图简化更像基础清理和铺路优化
