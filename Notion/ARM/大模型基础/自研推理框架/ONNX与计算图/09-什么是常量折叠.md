# 什么是常量折叠

## 1. 一句话定义

常量折叠（Constant Folding）就是：

> 如果一个子图的输入在编译期就都是常量，那就提前算出结果，用一个常量替代整段子图。

也就是把：

- 运行时计算

变成：

- 编译时提前计算

## 2. 它在大模型图里长什么样

一个典型例子是 Attention 的缩放因子。

原始图可能长这样：

```text
Const(128)
   ↓
Sqrt
   ↓
Div(1, x)
   ↓
Mul(QK^T)
   ↓
Softmax
```

这里：

- `128` 是固定常量
- `sqrt(128)` 也是固定值
- `1 / sqrt(128)` 也是固定值

所以这段就可以折叠成：

```text
MatMul(Q, K)
   ↓
Mul(Const(0.088388))
   ↓
Softmax
```

这样：

- `Const → Sqrt → Div` 这段都被提前算掉了

## 3. 为什么它很重要

常量折叠的收益主要有 3 类：

- 减少运行时计算
- 减少 kernel launch
- 减少中间 tensor 和访存

在端侧设备上，这类收益尤其明显，因为很多“小节点”本身的算力占比不大，但调度和 launch 成本并不低。

## 4. 常量从哪里来

很多人会问：为什么图里会有“常量输入”？

常见来源有下面几类。

### 4.1 模型参数

例如：

- 权重
- bias
- embedding table

它们在推理阶段本身就是常量 tensor。

### 4.2 模型结构中的固定参数

例如：

- head_dim
- Attention scaling 因子
- 一些固定的系数

这些值在模型结构确定后就是常量。

### 4.3 shape / layout 相关常量

很多 `Reshape / Expand / Unsqueeze / Slice / Concat` 会依赖固定 shape 张量。

这些 shape tensor 也是常量。

### 4.4 exporter 自动插入的常量

从 PyTorch 导出 ONNX 时，exporter 往往会生成大量：

- shape 常量
- index 常量
- layout 常量

所以图里天然会有很多 `Const` 节点。

### 4.5 mask / position 等静态数据

例如：

- 固定 attention mask
- 固定 position id
- 某些固定长度下的索引表

这些在特定场景下也可能提前折叠。

## 5. 再看一个 LLM 里的常见例子

有些位置编码或索引生成流程可能导出成：

```text
Range → Cast → Unsqueeze → Expand
```

如果序列长度固定，这整段都可能提前算成一个常量 position tensor。

也就是说：

```text
Const(position_ids)
```

直接替代原来的生成子图。

## 6. 常量折叠和冗余消除的关系

常量折叠之后，原图中往往会留下很多：

- 不再被使用的节点
- 断掉的中间边
- 无意义分支

所以常量折叠之后，通常还会接着做冗余消除。

## 7. 面试速记版

- 常量折叠是把“只依赖常量的子图”提前算掉
- 在 LLM 图里常见于 scaling、shape、mask、position 这类子图
- 常量折叠减少计算、launch 和访存
- 折叠后通常还要接着做冗余消除
