# 9.1 左值、右值、右值引用

## 左值和右值

- 左值（lvalue）
  - 有名字、通常可取地址、可复用
  - 常可出现在赋值号左边
- 右值（rvalue）
  - 通常是临时值、短命对象、表达式结果
  - 更适合被“移动”而不是被“拷贝”

常见例子：

```cpp
int x = 1;         // x 是左值
int y = x + 2;     // x + 2 是右值
std::string("hi"); // 临时对象，是右值
```

## 右值到底有什么用

右值的真正意义主要有两个：

1. 表示“短暂的临时对象”
2. 表示“可以安全转移资源的对象”

也就是说，右值让编译器和程序员知道：

- 这个对象很快就没用了
- 因而可以把它内部资源直接搬走，而不是深拷贝一份

## 右值的生命周期

- 右值临时对象通常在“完整表达式结束”时销毁
- 但若被 `const T&` 绑定，生命周期可被延长到该引用结束

例如：

```cpp
std::string("hello");               // 这一行结束后销毁
foo(std::string("abc") + "def");    // 整个函数调用表达式结束后销毁
const std::string& r = std::string("x"); // 生命周期延长到 r 结束
```

## 右值引用

- 右值引用写作 `T&&`
- 主要用于移动语义和完美转发
- 目的是避免不必要的深拷贝

例如：

```cpp
std::string&& rr = std::string("hello");
```

## 一个容易混的点

- “右值引用类型”不等于“右值表达式”
- 一个有名字的右值引用变量，本身是左值表达式

例如：

```cpp
std::string&& rr = std::string("hello");
foo(rr);            // rr 是左值
foo(std::move(rr)); // 这里才显式转成右值
```

## 转发引用 / 万能引用

只有在模板推导等特定场景里，`T&&` 才可能表示“转发引用”（也常被口语叫万能引用）：

```cpp
template <typename T>
void wrapper(T&& x);
```

这里的 `T&&` 不一定是纯右值引用，它可能同时接受左值和右值。

而下面这种就只是普通右值引用：

```cpp
int&& x = 1;
```

## 一句话总结

- 左值：可复用对象
- 右值：临时值 / 将亡值
- 右值引用：绑定右值，为移动语义和完美转发服务

# 9.2 四种类型转换

- `static_cast`
  - 常规类型转换，编译期可检查一部分错误
- `dynamic_cast`
  - 用于多态类型之间的安全向下转型，运行期检查
- `const_cast`
  - 去掉或添加底层 `const`
- `reinterpret_cast`
  - 按位重新解释，最危险，平台相关性强

## 四种转换的适用直觉

- 基本类型或已知安全转换：`static_cast`
- 多态层次安全向下转型：`dynamic_cast`
- 增删底层 `const`：`const_cast`
- 底层比特重解释：`reinterpret_cast`

## 面试里容易答错的点

- `dynamic_cast` 主要用于多态类型层次，不适合普通基本类型转换
- `static_cast` 可做向下转型，但只在你非常确定真实类型时才安全
- `reinterpret_cast` 能编译不代表语义安全

# 9.3 `std::move`

- `std::move` 本身不移动资源
- 它只是把一个表达式显式转成“可以被移动的右值形式”
- 真正发生移动的是后续的移动构造或移动赋值

```cpp
std::string s = "hello";
std::string t = std::move(s); // 这里真正可能发生资源转移
```

## 面试最稳的说法

- `std::move` 不动内存
- 它只是告诉编译器：“这个对象后续可以按右值处理”

## 为什么要有 `std::move`

- 命名对象默认都是左值
- 但有时候我们明知道它后面不会再用了
- 这时可以用 `std::move` 把它“标记”为可移动

例如：

```cpp
std::vector<std::string> v;
std::string s = "abc";
v.push_back(std::move(s)); // 比拷贝更可能触发移动
```

## 为什么需要移动语义

- 很多对象内部管理堆内存、文件句柄、锁等资源
- 若每次传递或返回都深拷贝，开销会很大
- 移动语义允许转移资源所有权，而不是重新复制整份资源

# 9.3.1 移动构造函数

移动构造函数通常形如：

```cpp
ClassName(ClassName&& other) noexcept;
```

## 核心作用

- 把资源所有权从临时对象或将亡值对象转移到新对象
- 避免深拷贝带来的额外开销

## 典型场景

- 函数返回临时对象
- 容器扩容搬运元素
- `std::move` 触发移动语义
- 完美转发保留右值属性时

## 常见使用它的库

- `std::vector`
- `std::string`
- 智能指针等资源管理类

# 9.3.2 拷贝构造和移动构造的区别

- 拷贝构造
  - 复制资源
- 移动构造
  - 转移资源所有权

## 典型差异

- 拷贝通常需要新分配资源
- 移动通常只搬指针/句柄等控制信息，代价更小

## 移动后对象怎么办

- 标准通常只要求“处于有效但未指定状态”
- 也就是它还能析构、还能重新赋值
- 但不要指望它还保留原值

# 9.4 `char` 和 C 风格字符串

## `char str[] = "hello"` 与 `char* str = "hello"` 的区别

```cpp
char str1[] = "hello";
const char* str2 = "hello";
```

- `str1` 是字符数组，数组内容可修改，实际存的是一份拷贝。
- `str2` 是指针，指向字符串字面量。
- 字符串字面量通常位于只读存储区，不能通过 `str2` 修改其内容。

## 注意

- 在现代 C++ 里，字符串字面量类型是 `const char[N]`，因此不要写成 `char* str = "hello"`。

# 9.5 指针和引用

## 区别

- 指针保存地址，本身是对象。
- 引用是别名，不是独立对象。
- 指针可以为空，可以改指向。
- 引用必须初始化，绑定后不能改绑。
- 指针可以多级，引用通常只作为一级别名使用。

## 什么时候用指针，什么时候用引用

- 可能为空、需要重新指向、处理数组衰减时，用指针。
- 表达“必然存在的别名”、避免拷贝时，优先用引用。
- 类对象参数传递通常优先用 `const T&`。
- 数组形参本质上会退化为指针。

## 类内定义引用成员

类里可以有引用成员，但要注意：

- 引用成员必须在初始化列表中初始化
- 引用一旦绑定，之后不能改绑
- 类通常需要自定义构造函数来完成引用成员初始化

例如：

```cpp
struct Holder {
    int& ref;
    Holder(int& x) : ref(x) {}
};
```

# 9.6 `strlen` 和 `sizeof` 的区别

## `sizeof`

- 运算符
- 编译期通常即可确定结果
- 求类型或对象占用的字节数
- 参数可以是类型、变量、表达式

## `strlen`

- C 标准库函数
- 运行期遍历字符直到 `'\0'`
- 返回字符串长度，不包含结尾空字符
- 参数必须是以 `'\0'` 结尾的字符串

## 典型例子

```cpp
char s[] = "abc";
sizeof(s);   // 4
strlen(s);   // 3
```

## 注意

- 指针变量做 `sizeof` 得到的是指针本身大小，不是它指向内容的长度。
- 对动态分配的内存，`sizeof(ptr)` 依然只是指针大小。

# 9.7 `#define` 和 `typedef`

- `#define` 是预处理替换，不做类型检查。
- `typedef` 是类型别名，受编译器类型系统约束。
- 现代 C++ 更推荐 `using`，可读性通常更好。

## 纠正一点

- `typedef` 不会“在静态存储区分配一份空间”
- 它只是类型别名，不创建对象，不分配存储

# 9.8 `constexpr` 和 `const`

- `const` 表示只读，不一定是编译期常量。
- `constexpr` 要求表达式可在编译期求值。
- `constexpr` 对象隐含 `const`，反过来不成立。

## 顶层 `const` 和底层 `const`

- 顶层 `const`
  - 限定对象本身不能改
- 底层 `const`
  - 限定通过该类型不能修改所指对象

例如：

```cpp
int x = 1;
const int* p1 = &x;  // 底层 const
int* const p2 = &x;  // 顶层 const
const int* const p3 = &x;  // 都有
```

# 9.8.1 常见 `const` 写法辨析

```cpp
const int* a;
int const* b;
const int c = 1;
int* const d = &x;
const int* const e = &x;
```

- `const int* a`
  - 指向常量的指针，不能通过 `a` 改值，可改指向
- `int const* b`
  - 和上面等价
- `const int c`
  - 常量，值不能改
- `int* const d`
  - 指针常量，指向不能改，但可通过它改值
- `const int* const e`
  - 指向常量的指针常量，指向和值都不能通过它改

# 9.9 函数指针和指针函数

- 函数指针：本质是指向函数的指针。
- 指针函数：本质是返回值为指针的函数。

例如：

```cpp
int* func(int);
int (*fp)(int);
```

# 9.10 C++ 常见传参方式

- 值传递：会拷贝实参
- 指针传递：可表示“可空”和“可修改指向对象”
- 引用传递：语义更直接
- 常量引用：大对象传参的常见首选
- 右值引用：用于移动语义和转发

## 几种方式的区别

- 值传递
  - 会拷贝实参
  - 不影响原对象
- 引用传递
  - 直接操作别名
  - 可避免拷贝
- 指针传递
  - 可表达“可空”
  - 需要显式解引用

## 面试建议

- 小对象可值传递
- 大对象常用 `const T&`
- 需要表达“可能为空”时常用指针

# 9.10.2 值传递、指针传递、引用传递的区别和效率

- 值传递
  - 拷贝一份实参
  - 不影响原对象
  - 大对象可能较慢
- 指针传递
  - 传的是地址值
  - 可空
  - 通过解引用访问目标
- 引用传递
  - 本质上是别名语义
  - 不需要对象拷贝
  - 通常比值传递更高效也更直观

## 常见结论

- 大对象优先 `const T&`
- 需要“可空”语义时常用指针

# 9.10.3 常量引用

- `const T&` 可以绑定左值
- 也可以绑定右值和临时对象
- 常用于避免大对象拷贝

## 面试一句话

- 常量引用最常见的价值是“避免拷贝 + 保证只读”

# 9.10.1 前置 `++i` 和后置 `i++`

## 语义区别

- `++i`
  - 先自增，再返回
- `i++`
  - 先保存旧值，再自增，返回旧值

## 对内置类型

- 结果差异主要体现在表达式值

## 对自定义迭代器/对象

- 前置 `++` 通常返回引用
- 后置 `++` 通常需要保存旧值并返回临时对象
- 因此前置 `++` 往往更高效

## 面试建议

- 在不需要旧值时，优先使用前置 `++`

# 9.11 浮点数为什么不建议直接 `==`

- 浮点数存在精度误差
- 很多十进制小数无法被二进制精确表示

常见做法：

```cpp
std::abs(a - b) < 1e-6
```

# 9.12 空类大小

- 空类通常大小为 1 字节
- 原因是不同对象需要有不同地址
- 若类含虚函数，通常还会多出一个虚表指针开销

# 9.13 静态链接和动态链接

- 静态链接：库代码在链接期拷贝进可执行文件
- 动态链接：运行时加载共享库

常见对比：

- 静态链接部署简单，但体积更大
- 动态链接便于共享和升级，但依赖运行环境

# 9.14 自动类型推导

- `auto` 让代码更简洁
- 适合复杂模板类型、迭代器类型
- 但不能滥用，避免损失可读性

## `auto` 的常见规则

- 必须有初始化器，编译器才能推导
- 一般会忽略顶层 `const`
- 一般不会保留引用，除非显式写成 `auto&` / `auto&&`

例如：

```cpp
const int x = 1;
auto a = x;   // int
auto& b = x;  // const int&
```

## `Debug` 和 `Release` 的区别

- `Debug`
  - 更便于调试
  - 通常包含调试符号
  - 优化较少或关闭优化
  - 运行速度和体积通常不如 `Release`
- `Release`
  - 面向发布
  - 更强调优化后的性能和体积
  - 调试信息通常更少

## 注意

- 不同编译器、构建系统和项目配置下，二者的具体差异会不同。
- 面试里一般答“是否开启优化、是否带调试信息、断言和日志策略是否不同”就够了。

# 9.15 指针数组和数组指针

## 指针数组

- 本质是数组
- 数组里的每个元素都是指针

```cpp
int a = 1, b = 2, c = 3;
int* arr[3] = {&a, &b, &c};
```

## 数组指针

- 本质是指针
- 指向整个数组

```cpp
int nums[3] = {1, 2, 3};
int (*p)[3] = &nums;
```

## 二维数组指针

例如：

```cpp
int a[2][3] = {{1, 2, 3}, {4, 5, 6}};
int (*p)[3] = a;
```

- `p` 指向“包含 3 个 `int` 的一维数组”
- `p + 1` 会移动到下一行

# 9.15.1 数组和指针的区别

- 数组是连续元素的集合
- 指针是保存地址的变量
- 数组名多数场景会退化为首元素指针，但二者不是同一个东西
- 数组不能整体赋值，指针可以重新指向别处
- `sizeof(数组)` 与 `sizeof(指针)` 含义不同

# 9.16 数组名和指针的区别

- 数组名在大多数表达式中会退化为首元素指针。
- 但数组名本身不是普通可修改指针。
- 数组名不能自增、自减、重新赋值。
- `sizeof(数组名)` 得到整个数组大小，而不是首元素指针大小。

当数组作为函数参数时：

- 会退化为指针
- 此时 `sizeof` 不能再得到原数组总大小

# 9.17 字符串指针变量、字符数组、字符指针数组

## 字符数组

```cpp

```

- 本地保存了一份字符序列

## 字符串指针

```cpp
const char* p = "hello";
```

- `p` 指向字符串字面量首地址

## 字符指针数组

```cpp
const char* words[] = {"hello", "world"};
```

- 数组元素是多个字符指针

## `cout` 打印 `char*`

- 若想打印地址，可转成 `const void*`。

# 9.18 C 风格字符串常用函数

## `strcpy`

- 拷贝字符串，遇到 `'\0'` 结束
- 目标空间必须足够，否则可能溢出

## `sprintf`

- 按格式写入字符缓冲区
- 容易溢出，实际开发更推荐 `snprintf`

## `memcpy`

- 按字节拷贝指定长度
- 不关心数据类型和是否存在 `'\0'`
- 适合二进制块、结构体、数组等原始内存复制

## 区别总结

- `strcpy`：面向字符串
- `sprintf`：面向格式化输出
- `memcpy`：面向原始内存块

## `memcpy` 和 `strcpy` 的区别

- `strcpy`
  - 以 `'\0'` 为结束标志
  - 只适合 C 字符串
- `memcpy`
  - 按指定字节数拷贝
  - 可处理二进制块、数组、结构体等
  - 不要求数据里有 `'\0'`
  - 若源和目标区间重叠，应改用 `memmove`

# 9.19 `string` 和 `char*` 的区别

- `std::string` 是标准库类，`std::string` 实际上是 `std::basic_string<char>` 的类型别名。
- `char*` 只是字符地址，不负责容量、长度和内存管理。

## `std::string` 的优势

- 自动管理内存
- 自动记录长度
- 支持拼接、查找、截取、比较等操作
- 更安全，接口更丰富

## 实现理解

- 一般内部维护：
  - 字符缓冲区
  - 当前长度
  - 当前容量
- 扩容策略由具体实现决定，不是标准强制规定“每次一定扩两倍”。

# 9.20.1 写一个比较大小模板

最简单的函数模板可以这样写：

```cpp
template <typename T>
const T& my_max(const T& a, const T& b) {
    return (a < b) ? b : a;
}
```

## 面试可补一句

- 这里要求类型 `T` 支持 `<` 运算。
- 用 `const T&` 可以避免不必要拷贝。

# 9.21 迭代器

## 概念

- 迭代器是访问容器元素的一种统一抽象
- 作用类似“泛化后的指针”

## 指针和迭代器的异同

- 指针主要适合连续内存结构
- 迭代器可适配连续或非连续容器
- 很多容器并不是连续存储，例如 `list`、`map`

## 常见类型

- 输入迭代器
- 输出迭代器
- 前向迭代器
- 双向迭代器
- 随机访问迭代器

## `begin/end` 与 `cbegin/cend`

- 非常量容器可返回可写迭代器
- 常量容器返回只读迭代器
- `cbegin/cend` 明确返回常量迭代器

# 9.21.1 各种指针类型

常见可以从这几个维度回答：

## 按是否可修改

- 指向常量的指针：`const int* p`
- 指针常量：`int* const p`
- 指向常量的指针常量：`const int* const p`

## 按用途

- 空指针：`nullptr`
- 野指针
- 悬空指针
- 函数指针
- `void*`

## `void*`

- 可保存任意对象地址
- 不能直接解引用
- 使用前通常需要显式转换

# 9.21.2 函数指针

## 什么是函数指针

- 指向函数入口的指针变量

## 定义方式

```cpp
int func(int x);
int (*fp)(int) = func;
```

## 使用场景

- 回调函数
- 策略分发
- C 风格接口表
- 简单状态机或插件式设计

# 9.21.3 使用指针时需要注意什么

- 定义后尽快初始化
- 动态分配后检查结果是否符合预期
- 释放后及时置空
- 避免越界访问
- 避免返回局部变量地址
- 保证申请和释放成对
- 能用智能指针和容器时尽量不用裸指针

# 9.21.4 C++ 和 Python 的区别

- C++ 通常先编译再运行，Python 通常由解释器执行
- C++ 更接近底层，性能通常更高
- Python 开发效率更高，语法更简洁
- C++ 手动资源管理能力更强，Python 常依赖垃圾回收
- C++ 静态类型更强，Python 更偏动态类型

## 面试一句话

- C++ 偏性能和系统能力
- Python 偏开发效率和生态便利

# 9.21.7 可变参数模板

- 用 `template <typename... Args>` 表示模板参数包
- 用 `Args... args` 表示函数参数包
- 适合写通用转发、日志封装、工厂函数等代码

例如：

```cpp
template <typename... Args>
void print_all(const Args&... args) {}
```

## 面试常答点

- 比 C 风格可变参数更安全
- 保留了类型信息
- 常和递归展开或折叠表达式一起使用

# 9.21.8 Lambda 表达式

Lambda 本质上是一个匿名函数对象。

```cpp
auto f = [x](int y) { return x + y; };
```

## 组成部分

- 捕获列表 `[]`
- 参数列表 `()`
- 返回类型（可省略）
- 函数体 `{}`

## 常见捕获方式

- `[=]`
  - 按值捕获
- `[&]`
  - 按引用捕获
- `[this]`
  - 捕获当前对象指针

## 注意

- Lambda 默认生成一个带 `operator()` 的闭包对象
- 默认情况下，按值捕获的变量在 `operator()` 中不可修改；若要修改可加 `mutable`

# 9.21.9 闭包、`std::function`、完美转发

## 闭包是什么

- 闭包可以理解为带状态的可调用对象
- Lambda 捕获外部变量后，编译器通常会把这些捕获变量变成闭包对象的成员

## `std::function`

- `std::function` 是通用可调用对象包装器
- 可统一保存普通函数、函数指针、Lambda、仿函数、`bind` 结果

例如：

```cpp
std::function<int(int, int)> op = [](int a, int b) { return a + b; };
```

## `std::forward`

- `std::forward` 用于完美转发
- 主要在转发引用场景中保留参数原本的左值/右值属性

```cpp
template <typename T>
void wrapper(T&& x) {
    func(std::forward<T>(x));
}
```

## `std::move` 和 `std::forward` 的区别

- `std::move`
  - 无条件把对象转成右值
- `std::forward`
  - 按模板推导结果保留原始值类别

## 一句话

- `move` 是主动把它当右值
- `forward` 是按它原来的样子转交出去

# 9.21.5 仿函数 / 函数对象

仿函数本质上是重载了 `operator()` 的类对象。

例如：

```cpp
struct Add {
    int operator()(int a, int b) const { return a + b; }
};
```

## 作用

- 像函数一样调用
- 又能像对象一样携带状态
- 常用于 STL 算法、自定义比较器、回调策略

# 9.22 `stack`、`queue`、`priority_queue`、红黑树

## `stack`

- 容器适配器
- 默认底层容器通常是 `deque`

## `queue`

- 容器适配器
- 默认底层容器通常也是 `deque`

## `priority_queue`

- 本质上通常基于堆
- 默认底层容器通常是 `vector`

## 红黑树

- `map`、`set`、`multimap`、`multiset` 通常基于红黑树实现
- 特点是近似平衡，查找、插入、删除复杂度为 `O(log n)`

## 工程补充

- `epoll` 的就绪事件管理不是简单等同于“红黑树”，更准确地说，Linux 内核中会用红黑树管理感兴趣的文件描述符，同时配合链表组织就绪事件。
