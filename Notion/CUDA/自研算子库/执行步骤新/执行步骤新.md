2.prefill和decode算子差异

两者的差异只有attention的计算 kvcache的访问两点

| 算子 | Prefill | Decode | 是否语义不同 |
| --- | --- | --- | --- |
| Embedding | ✅ | ✅ | ❌ |
| Rotary / Positional | ✅ | ✅ | ❌ |
| **RMSNorm（block）** | ✅ | ✅ | ❌ |
| **QKV Linear** | ✅ | ✅ | ❌ |
| **q_norm** | ✅ | ✅ | ❌ |
| **k_norm** | ✅ | ✅ | ❌ |
| **Attention** | ⚠️ 不同 | ⚠️ 不同 | ✅ |
| **KV Cache** | ⚠️ 写为主 | ⚠️ 读+写 | ✅ |
| Output Linear | ✅ | ✅ | ❌ |
| FFN (SwiGLU) | ✅ | ✅ | ❌ |

一、Prefill vs Decode 的本质差异

| 维度 | Prefill | Decode |
| --- | --- | --- |
| Q shape | `[B, S, H, d]` | `[B, 1, H, d]` |
| K/V | `[B, S, H, d]` | KV cache `[B, T, H, d]` |
| 核心瓶颈 | **计算（GEMM）** | **访存（KV cache）** |
| 是否可 Flash | 是 | 否（结构不同） |
| kernel 类型 | tiled block | warp-level |
| 优化重点 | 算子融合 | 内存布局 + cache 访问 |

| 算子 | 使用阶段 | 设计思路 |
| --- | --- | --- |
| FlashAttention (Prefill) | Prefill | tiled QK + online softmax |
| DecodeAttention (1×T) | Decode | warp softmax + streaming KV |

### 4️⃣ 算子融合策略完全不同

### Prefill

- 融合重点：
    - QK GEMM + softmax + PV
- 目的是减少中间矩阵

### Decode

- 融合重点：
    - Q·K + softmax + V 累加
- 目的是：
    - 少访存
    - warp 内完成

👉 同样是“attention”，**融合目标都不一样**。
