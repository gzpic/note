# C 语言基础

## 1. C 语言的特点

- 语法简洁、接近底层，适合描述硬件相关逻辑。
- 可移植性强，能在多种体系结构上运行。
- 运行效率高，编译后几乎没有额外运行时负担。
- 支持指针、位运算、结构体、函数指针等底层能力。
- 适合系统编程、驱动开发和嵌入式开发。

### 面试回答建议

- C 最大优势是“高效、可控、可移植、接近硬件”。
- 但它的抽象能力和安全性相对较弱，复杂项目里更依赖编码规范和工程经验。

## 2. 为什么嵌入式里 C 用得多

- 可以方便访问寄存器和内存映射 IO。
- 代码体积小、执行效率高，适合资源受限环境。
- 工具链成熟，几乎所有 MCU 平台都支持 C。
- 比汇编更容易维护、移植和表达复杂逻辑。

### 和汇编的分工

- 汇编：
  - 启动代码
  - 中断入口
  - 上下文切换
  - 极少量对时序极敏感的代码
- C：
  - 驱动逻辑
  - 协议处理
  - 状态机
  - 业务功能

## 3. 常见内存布局

编译后的程序通常会分布在以下区域：

- `.text`：代码段，存放机器指令
- `.rodata`：只读数据，例如字符串常量
- `.data`：有初始值的全局/静态变量
- `.bss`：未初始化或初始化为 0 的全局/静态变量
- `Heap`：动态分配区
- `Stack`：函数调用现场和局部变量

| 段名 | 存放内容 | 是否可修改 | Flash 是否占空间 | RAM 是否占空间 | 启动时动作 |
| --- | --- | --- | --- | --- | --- |
| `.text` | 程序代码、中断向量等 | 否 | 是 | 通常否 | 直接执行 |
| `.rodata` | 字符串常量、只读常量 | 否 | 是 | 视平台而定 | 通常无需初始化 |
| `.data` | 有初值的全局/静态变量 | 是 | 是 | 是 | 从 Flash 拷贝到 RAM |
| `.bss` | 无初值或 0 初值的全局/静态变量 | 是 | 否 | 是 | 启动时清零 |

## 4. 栈、寄存器与函数调用

- 函数执行期间主要使用栈空间保存返回地址、局部变量和保存寄存器。
- 编译器会优先把小型、热点局部变量放进寄存器。
- 数组、结构体、较大的局部变量通常更可能放在栈上。

### ARM 调用约定常见记忆点

- `R0-R3`：参数和返回值
- `R4-R11`：常作为被调用者保存寄存器
- `R13`：`SP` 栈指针
- `R14`：`LR` 返回地址
- `R15`：`PC` 程序计数器

## 5. 两个常见易错点

- `const` 变量不一定都在只读区，是否进入 `.rodata` 取决于作用域、是否取地址、编译器实现。
- “局部变量都在栈上” 也不绝对，编译器可能把它们分配到寄存器中。
