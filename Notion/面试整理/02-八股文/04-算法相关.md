# 算法相关

## 1. Transformer 主流程

从 `[[ARM/大模型基础/大模型推理核心/公式推导]]` 提炼出的核心点：

- 输入 token 先做 embedding，得到 `[T, D]`
- 每层基本结构：
  - LayerNorm
  - Q/K/V 线性映射
  - Self-Attention
  - 残差
  - LayerNorm
  - FFN/MLP
  - 残差
- 最后：
  - `h_final @ W_vocab -> logits`
  - `softmax -> next token`

## 2. Attention

已提炼出的点：

- `Q = hWQ, K = hWK, V = hWV`
- Attention 的核心是 `softmax(QK^T / sqrt(d))V`
- Prefill 阶段一般是整段 attention
- Decode 阶段一般是 `1 x T` attention，核心负担变成读历史 KV

来源：

- `[[ARM/大模型基础/大模型推理核心/公式推导]]`
- `[[CUDA/自研算子库/执行步骤新/执行步骤新]]`

## 3. FFN / SwiGLU

从 `[[CUDA/自研算子库/执行步骤新/swiglu]]` 提炼出的核心点：

- 常见形式是门控分支 + 上投影分支
- 工程里经常把 `Wgate` 和 `Wup` 合并成一次 GEMM
- `SiLU/Swish` 往往融合到 epilogue
- 本质可理解为：
  - GEMM
  - 激活/门控
  - elementwise
  - down projection

## 4. Online Softmax / FlashAttention 思想

从 `[[CUDA/自研算子库/执行步骤新/online softmax的计算]]` 提炼出的核心点：

- 维护运行中的最大值 `m`
- 维护分母累计值 `d`
- 每来一块就对旧分子和旧分母统一重缩放
- 这样可以避免完整保存巨大中间矩阵

## 5. Prefill vs Decode 的算法差异

从 `[[CUDA/自研算子库/执行步骤新/执行步骤新]]` 提炼出的核心点：

- Prefill：
  - `Q` shape 是 `[B, S, H, d]`
  - 更偏计算密集
  - 适合 FlashAttention 风格
- Decode：
  - `Q` shape 是 `[B, 1, H, d]`
  - 更偏访存密集
  - 关键在 KV cache 读取和 warp 级 softmax

## 6. KV Cache

从 `[[ARM/大模型基础/自研推理框架/KvCache/KvCache]]` 提炼出的核心点：

- 不同系统差异主要在：
  - 数据结构
  - 复用粒度
  - 显存不足时的回收策略
- 代表思路：
  - vLLM：block/page 管理
  - SGLang：前缀树
  - TensorRT-LLM：block/page + offload

## 8. Softmax 与溢出问题

- softmax 实现时通常先减去最大值：
- `softmax(x) = exp(x - max(x)) / sum(exp(x - max(x)))`
- 这样做是为了避免 `exp` 溢出。

## 9. L1 范数和 L2 范数

- L1
  - 倾向于产生稀疏解
  - 常用于特征选择
  - 对异常值更鲁棒
- L2
  - 倾向于让参数整体更平滑
  - 常用于抑制过拟合
  - 数值优化更稳定

## 10. CNN、RNN、GNN 的区别

- CNN
  - 擅长提取局部空间特征
  - 常用于图像
- RNN
  - 擅长建模时序依赖
  - 常用于序列数据
- GNN
  - 擅长图结构上的邻居消息传递
  - 常用于推荐、社交网络、分子图

## 11. 数据不平衡怎么办

- 欠采样多数类
- 过采样少数类
- 数据增强
- 加权损失函数
- 调整评估指标，不只看 accuracy

## 12. 梯度爆炸

常见原因：

- 初始化不当
- 网络太深
- 学习率过大
- 激活函数或损失函数导致梯度不稳定

常见做法：

- 梯度裁剪
- 更稳定的初始化
- 合理归一化
- 调整学习率
- 正则化

## 13. DFS 递归改迭代

- 核心是用显式栈模拟递归调用栈。
- 这样可以避免深递归导致栈溢出。
- 工程里也更方便控制状态恢复和遍历顺序。

## 14. Python 可变对象和不可变对象

- 不可变对象：
  - `int`
  - `float`
  - `str`
  - `tuple`
- 可变对象：
  - `list`
  - `dict`
  - `set`

面试简答：

- 不可变对象修改本质是创建新对象。
- 可变对象原地改内容时，外部引用通常也会看到变化。

## 15. BPE 和 WordPiece

- BPE
  - 反复合并出现频率最高的相邻子词对
  - 减少 OOV
- WordPiece
  - 也是子词方法
  - 更强调基于概率或语言模型目标选 token

## 16. 困惑度

- 困惑度衡量模型对下一个词预测的不确定性。
- 越低通常越好。
- 最理想情况接近 1。

## 17. 面试补充题

- 链表和顺序表区别：
  - 顺序表随机访问快
  - 链表插入删除快
- 思维链：
  - 让模型显式分步推理，复杂任务效果通常更好
