# 内存大小预估

## 1. 部署时要拆成哪几块

从 `[[CUDA/模型咋部署]]` 提炼：

- 模型权重占用
- KV Cache 占用
- workspace / 临时 buffer
- activation / 中间 tensor
- runtime 额外开销

## 2. 权重估算

通用近似：

- FP16：`参数量 × 2 bytes`
- BF16：`参数量 × 2 bytes`
- INT8：`参数量 × 1 byte`
- INT4：`参数量 × 0.5 byte`

举例：

- `7B` 模型 FP16 权重大约 `14 GB`
- `7B` 模型 INT4 权重大约 `3.5 GB`

## 3. 总显存估算的答题框架

`总显存 ≈ 权重 + KV Cache + activation/workspace + 碎片/冗余`

如果是端侧设备，还要额外提：

- UMA/统一内存迁移风险
- 运行时 buffer 预留
- 长时间运行的温度和降频影响

## 4. 面试时容易被追问的点

- 为什么理论算够了，实际还是 OOM
- 为什么 batch 不大，显存还是爆
- 为什么只剩一个请求也申请失败

这类问题通常要补充：

- allocator 碎片
- block/page 对齐损耗
- 历史 KV 未及时回收
- workspace 峰值和 steady-state 不一致
