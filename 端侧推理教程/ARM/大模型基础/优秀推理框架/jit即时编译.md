### 🧠 JIT（即时编译） vs 解释执行（Interpreter Execution）

| 对比维度 | **JIT（Just-In-Time Compilation）** | **解释执行（Interpreter Execution）** |   |
| --- | --- | --- | --- |
| 🕐 **执行时机** | 程序运行时动态编译成机器码 | 运行时逐行解释字节码 |   |
| 🧱 **编译层级** | Python/IR → LLVM IR → **机器码（native code）** | Python → **字节码（bytecode）**，由解释器执行 |   |
| ⚙️ **运行机制** | 边执行边编译，生成底层可直接运行的二进制 | 每行语句都由解释器实时解析与调度 |   |
| 💾 **缓存行为** | 可缓存已编译的机器码，后续直接调用 | 每次执行都需重新解释字节码 |   |
| 🚀 **性能表现** | 快得多，可接近 C/C++ 水平（取决于优化） | 较慢，每条语句都需解释调度 |   |
| 🧮 **优化能力** | 编译器可做循环展开、向量化、内联、常量折叠等 | 解释器无法做复杂优化 |   |
| 🔧 **典型应用** | Numba、Triton、PyPy、TorchInductor、TVM | CPython、普通 Python、早期 JavaScript |   |
| 🔄 **灵活性** | 高，可根据运行时输入动态生成专用机器码 | 最高，无需编译，随写随跑 |   |
| 🧩 **适合场景** | 性能关键代码（数值计算、GPU kernel） | 逻辑控制、开发调试 |   |
| 🧠 **运行实质** | “即时编译+原生执行” | “逐条解释执行” |   |
| 🏁 **输出形式** | 真正的机器码（CPU/GPU 指令） | 字节码，由解释器翻译执行 |   |

解释执行：运行时逐条读源码 → 执行字节码；

JIT 编译：运行时编译热点代码 → 执行机器码；

### 🔬 举例对应

| 技术 | 类型 | 输出 |
| --- | --- | --- |
| CPython | 解释执行 | 字节码 |
| PyPy | JIT | x86 机器码 |
| Numba | JIT | LLVM → CPU 机器码 |
| Triton | JIT | LLVM → PTX → GPU SASS |
| C/C++ (g++) | AOT | 机器码 |
