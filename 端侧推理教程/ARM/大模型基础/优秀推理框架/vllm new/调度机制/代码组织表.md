## 🧱 1️⃣ `PreemptionMode(enum.Enum)`

**作用：**

定义调度器在内存不足或需要让出资源时的两种抢占模式。用于 `Scheduler` 中决定是“换出内存”还是“重新计算”。

| 函数/属性 | 作用说明 |
| --- | --- |
| `SWAP` | 抢占时将 KV 缓存从 GPU 换出到 CPU（可稍后再加载回来）。 |
| `RECOMPUTE` | 抢占时直接丢弃缓存，下次重新计算。 |

---

## 🧱 2️⃣ `SchedulingBudget`

**作用：**

封装调度器的一次调度预算控制逻辑，包括 token 数量、序列并发数量等。

调度前后会动态更新，防止超出 GPU 容量。

| 函数/属性 | 作用说明 |
| --- | --- |
| `can_schedule(num_new_tokens, num_new_seqs)` | 判断是否有足够预算调度新的 token 或序列。 |
| `remaining_token_budget()` | 获取剩余 token 预算。 |
| `add_num_batched_tokens(req_id, num_batched_tokens, num_cached_tokens=0)` | 增加该请求的 token 计数。 |
| `subtract_num_batched_tokens(req_id, num_batched_tokens)` | 减少 token 预算。 |
| `add_num_seqs(req_id, num_curr_seqs)` / `subtract_num_seqs(req_id, num_curr_seqs)` | 调整活跃序列数。 |
| `num_batched_tokens` | 当前批次内已使用的 token 数。 |
| `num_curr_seqs` | 当前调度批中的序列数。 |
| `num_cached_tokens` | 当前批次中可重用缓存的 token 数。 |

---

## 🧱 3️⃣ `ScheduledSequenceGroup`

**作用：**

表示“已被选中进入本次调度”的一个序列组。包含序列对象引用和要处理的 token 块数。

| 属性 | 作用说明 |
| --- | --- |
| `seq_group` | 调度器正在处理的序列组对象。 |
| `token_chunk_size` | 本轮要计算的 token 数（prefill 时可能 >1，decode 时为 1）。 |

---

## 🧱 4️⃣ `SchedulerOutputs`

**作用：**

封装一次调度的输出结果，告诉执行器：

- 哪些序列要运行；
- 哪些块要 swap/copy；
- 哪些任务被忽略。

| 函数 | 作用说明 |
| --- | --- |
| `__post_init__()` | 初始化后检查合法性，并按 LoRA ID 排序。 |
| `is_empty()` | 判断结果是否为空。 |
| `_sort_by_lora_ids()` | 按 LoRA ID + 请求 ID 排序。 |
| `lora_requests` | 获取调度中涉及的 LoRA 请求集合。 |

---

## 🧱 5️⃣ `SchedulerRunningOutputs`

**作用：**

描述“运行中队列”在本次调度的选中结果，包含 decode/prefill 两类任务，以及抢占/换出信息。

| 函数 | 作用说明 |
| --- | --- |
| `create_empty()` | 创建一个空的运行调度结果。 |

---

## 🧱 6️⃣ `SchedulerSwappedInOutputs`

**作用：**

描述“已被 swap 出的任务”重新被换入 GPU 时的调度结果。

包括哪些块要载入，哪些无效请求被忽略。

| 函数 | 作用说明 |
| --- | --- |
| `create_empty()` | 创建空 swap-in 结果。 |

---

## 🧱 7️⃣ `SchedulerPrefillOutputs`

**作用：**

描述“等待队列”中被选中执行 prefill（提示阶段）的任务。

包括要启动的序列组和忽略的请求。

| 函数 | 作用说明 |
| --- | --- |
| `create_empty()` | 创建空 prefill 调度结果。 |

---

## 🧱 8️⃣ `PartialPrefillMetadata`

**作用：**

用于 **chunked prefill 并行** 策略。

记录当前已有多少“部分 prefill”在执行、多少是长任务、哪些还可以继续并发。

能避免长请求阻塞短请求。

| 函数 | 作用说明 |
| --- | --- |
| `can_schedule(seq_group)` | 判断是否还能并发调度该 prefill。 |
| `maybe_increment_partial_prefills(seq_group)` | 若是长请求则更新 long_prefills 计数。 |
| `from_queues(running, waiting, scheduler_config)` | 根据队列状态生成新的统计信息对象。 |

---

## 🧱 9️⃣ `Scheduler`

**作用：**

⭐ **核心调度器类**，负责整个推理过程中的请求队列管理、KV 内存分配、chunked prefill 策略、LoRA 管理、抢占与资源回收。

是 vLLM 连续批次调度（Continuous Batching）和异步 decode 的核心组件。

---

### 🔹 初始化与配置

| 函数 | 作用说明 |
| --- | --- |
| `__init__()` | 初始化调度器（包括 BlockSpaceManager、LoRA 配置、缓存池、队列）。 |
| `next_cache_id` | 双缓冲缓存切换索引。 |
| `lora_enabled` | 判断是否启用 LoRA 适配。 |
| `num_decoding_tokens_per_seq` | 每个序列 decode 阶段生成 token 数（默认 1）。 |

---

### 🔹 队列与任务管理

| 函数 | 作用说明 |
| --- | --- |
| `add_seq_group(seq_group)` | 把新请求加入等待队列。 |
| `_add_seq_group_to_running(seq_group)` / `_add_seq_group_to_swapped(seq_group)` | （测试用）直接加入运行或 swap 队列。 |
| `abort_seq_group(request_id, seq_id_to_seq_group=None)` | 根据请求 ID 强制终止并释放所有相关资源。 |
| `_free_seq_group_cross_attn_blocks(seq_group)` | 释放 encoder-decoder 的交叉注意力缓存。 |
| `has_unfinished_seqs()` | 是否仍有活跃序列。 |
| `get_num_unfinished_seq_groups()` | 当前未完成序列组数。 |
| `get_prefix_cache_hit_rate(device)` | 返回 prefix cache 命中率。 |
| `reset_prefix_cache(device)` | 清空 prefix cache。 |
| `get_and_reset_finished_requests_ids()` | 取出并清空已完成的请求 ID。 |

---

### 🔹 调度核心（Prefill / Decode / Swap）

| 函数 | 作用说明 |
| --- | --- |
| `_schedule_running()` | 调度运行态任务，包含 decode 阶段执行、chunked prefill 继续调度及抢占逻辑。 |
| `_schedule_swapped()` | 调度被 swap 出的任务重新换入。 |
| `_schedule_prefills()` | 调度等待中的 prefill 请求。 |
| `_schedule_priority_preemption()` | 按优先级进行强制抢占。 |
| `_schedule_default()` | 默认调度：prefill 优先，其次 decode。 |
| `_schedule_chunked_prefill()` | Chunked Prefill 调度策略（prefill + decode 并行）。 |
| `_schedule()` | 根据配置自动选择调度模式。 |
| `_order_finishing_prefills_first()` | 让即将完成的 prefill 排在前面（防止 decode 饥饿）。 |

---

### 🔹 抢占与内存换入换出

| 函数 | 作用说明 |
| --- | --- |
| `_preempt(seq_group, blocks_to_swap_out)` | 通用抢占入口，决定使用 SWAP 或 RECOMPUTE。 |
| `_preempt_by_recompute(seq_group)` | 丢弃缓存、等待重新计算。 |
| `_preempt_by_swap(seq_group, blocks_to_swap_out)` | 把序列块换出到 CPU。 |
| `_swap_in(seq_group, blocks_to_swap_in)` | 将块从 CPU 换入 GPU。 |
| `_swap_out(seq_group, blocks_to_swap_out)` | 将块从 GPU 换出 CPU。 |

---

### 🔹 调度主入口

| 函数 | 作用说明 |
| --- | --- |
| `schedule()` | 主调度接口。整合各阶段结果，生成 `SequenceGroupMetadata`，返回调度输出与异步标志。 |

---

### 🔹 底层内存与序列控制

| 函数 | 作用说明 |
| --- | --- |
| `fork_seq(parent_seq, child_seq)` | 在 BlockManager 中复制父序列（分支生成）。 |
| `free_seq(seq)` | 释放序列占用的块。 |
| `remove_seq_from_computed_blocks_tracker(seq_group, status)` | 删除该序列组相关的块追踪记录。 |
| `_remove_seq_from_computed_blocks_tracker(seq)` | 实际执行追踪清除。 |
| `_free_finished_seqs(seq_group)` | 释放序列组中所有已完成的序列块。 |
| `_free_finished_seq_group(seq_group)` | 完全释放序列组及 cross-attn 缓存。 |
| `free_finished_seq_groups()` | 遍历并清理所有已完成任务及异步停止任务。 |
| `_allocate_and_set_running(seq_group)` | 为序列分配内存块并切换状态为 RUNNING。 |
| `_append_slots(seq_group, blocks_to_copy, enable_chunking)` | 为序列组追加新 block（用于持续生成）。 |

---

### 🔹 辅助与策略函数

| 函数 | 作用说明 |
| --- | --- |
| `_passed_delay(now)` | 控制 prefill 调度延迟，使等待队列积累更多请求。 |
| `_get_prompt_limit(seq_group)` | 返回单条请求允许的最大 prompt 长度。 |
| `_get_priority(seq_group)` | 获取任务优先级（priority, arrival_time）。 |
| `_get_num_lookahead_slots(is_prefill, enable_chunking)` | 计算 speculative decoding 所需额外 slot 数量。 |
| `_get_num_new_uncached_and_cached_tokens(...)` | 计算当前任务要计算的 token 数（未缓存与缓存分开统计）。 |
| `_chunk_new_tokens_to_schedule(...)` | 在 chunked prefill 模式下按预算分块 token。 |
| `_can_append_slots(seq_group, enable_chunking)` | 判断是否能追加新的缓存块。 |
| `_allow_async_output_proc(seq_group)` | 判断当前序列是否可异步输出处理。 |

💡 **整体结构总览图：**

```plain text
PreemptionMode          → 抢占模式枚举
SchedulingBudget        → 控制 token/序列预算
ScheduledSequenceGroup  → 单次调度的任务单元
SchedulerOutputs        → 调度结果
SchedulerRunningOutputs → 运行态调度结果
SchedulerSwappedInOutputs → Swap 队列调度结果
SchedulerPrefillOutputs → Prefill 队列调度结果
PartialPrefillMetadata  → 并发 Prefill 统计
Scheduler               → ⭐ 核心调度器类
   ├── 初始化与配置
   ├── 队列与任务管理
   ├── 调度核心
   ├── 抢占与内存
   ├── 主入口 schedule()
   ├── 底层内存管理
   └── 辅助控制逻辑


```
