# 推理里哪些参数能量化

## 1. 推理阶段主要有哪些张量

从推理框架 / 算子视角看，大模型推理里常见张量可以分成 5 类：

- 模型权重
- 激活
- KV Cache
- Embedding / Norm 参数
- 中间临时 buffer

更关键的问题不是“有没有这些张量”，而是：

> 哪些张量值得量化，哪些张量量化收益高，哪些张量量化风险大。

## 2. 模型权重：量化主战场

典型权重包括：

- `token_embedding.weight`
- Attention 的 `Wq / Wk / Wv / Wo`
- FFN 的 `up / gate / down`
- Norm 的缩放参数，例如 RMSNorm 的 `gamma`

其中最适合量化的是线性层权重：

- QKV projection
- output projection
- FFN 的各层 Linear

常见方案：

- `INT8`
- `INT4`
- 粒度上多用 per-channel 或 per-group

一句话：

> 权重量化成熟度最高、收益最大，几乎是必做项。

## 3. 激活：不是不能量，而是难量

激活包括：

- Linear 输出
- FFN 中间态
- Attention 相关中间结果

它们的特点是：

- runtime 动态变化
- 与 token / sequence 强相关
- 分布不稳定

所以激活量化难点很大。

### 3.1 为什么激活值比权重更难量化

可以直接记一句：

> 权重是静态参数，激活是随输入不断变化的动态信号，所以激活更难用一套固定低比特去稳定表示。

核心原因主要有 5 个：

- 权重是离线固定的，激活是在线变化的
- 激活分布会随输入、token 位置、上下文长度变化
- 激活更容易出现 outlier，量化区间容易被极端值拉大
- 激活量化误差会继续传到下一层，累积更明显
- 某些模块比如 attention score、Softmax 前后、Norm 附近特别敏感

更直观点说：

- 权重量化更像“压缩一个已经看清楚的常量表”
- 激活量化更像“实时压缩一个分布一直在抖动的信号”

所以工程上常见现象是：

- `W4A16` 比较常见
- `W8A8` 需要更认真做校准
- 再往更低 bit 做 activation quant，精度风险会明显上升

### 3.2 激活为什么容易被 outlier 卡住

很多层的激活分布不是均匀的，而是：

- 大部分值集中在较小范围
- 少量值特别大

如果量化时为了容纳这些大值，把整体量化范围拉宽，就会导致：

- 常见小值分辨率下降
- 舍入误差增大
- 最终精度下降

这也是为什么很多量化方法都会重点处理 activation outlier。

特别敏感的部分通常包括：

- `QK^T` score
- Softmax 前后

这些位置通常不建议激进量化。

工程上更常见的做法是：

- 激活保 `FP16`
- 或借助 SmoothQuant 把难点更多转移给权重
- 或只在相对不敏感的位置做 `INT8 activation`

## 4. KV Cache：端侧第二主战场

KV Cache 在 decode 阶段会持续增长，形状通常接近：

```text
K / V: [B, H_kv, S, D]
```

它的特点是：

- 占显存大
- 访存压力重
- 在长上下文下越来越关键

所以 KV Cache 常常值得量化，常见路线有：

- `INT8 KV`
- 更保守的 per-head / per-channel scale

一句话：

> 如果权重是第一量化对象，KV Cache 往往是端侧的第二量化对象。

## 5. Embedding 和 Norm：收益有，但不是重点

### 5.1 Embedding

Embedding 通常可以量化，但一般不是首要优化点：

- 可以跟随权重量化
- 单独优化收益通常不如 Linear 权重明显

### 5.2 Norm 参数

例如 RMSNorm / LayerNorm 中的缩放参数，通常更偏向保留在 `FP16`：

- 参数量小
- 数值稳定性更重要
- 量化收益有限

## 6. 中间临时 buffer：通常不值得动

例如：

- Softmax buffer
- reduction 中间值
- 临时 GEMM buffer

这类张量：

- 生命周期短
- 数值敏感
- 量化收益有限

通常不建议专门量化。

## 7. 速查表

| 类别 | 是否建议量化 | 常见精度 | 备注 |
| --- | --- | --- | --- |
| Weights | 是 | INT8 / INT4 | 主战场 |
| Embedding | 可 | INT8 | 跟随权重 |
| Activations | 有条件 | FP16 / INT8 | 难点大 |
| KV Cache | 有条件 | INT8 | 端侧收益高 |
| Norm 参数 | 通常不建议 | FP16 | 保稳定 |
| Softmax / 临时 buffer | 不建议 | FP16 | 敏感 |

## 8. 面试速记版

- 权重一定是量化主战场
- 激活可以量，但比权重难很多
- KV Cache 是端侧第二主战场
- Norm 和 Softmax 一般优先保高精度
