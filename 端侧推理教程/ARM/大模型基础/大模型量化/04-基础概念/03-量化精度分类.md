# 量化精度分类

## 1. 按 bit 数分类

量化精度最常见的分类方式，就是看每个数值用多少 bit 表示。

| 精度 | 位宽 | 常见场景 |
| --- | --- | --- |
| FP32 | 32 bit | 训练、基准精度 |
| FP16 | 16 bit | GPU 推理主流 |
| BF16 | 16 bit | 训练、部分推理 |
| INT8 | 8 bit | 工业部署主流 |
| INT4 | 4 bit | 端侧 / 显存敏感场景 |
| INT3 | 3 bit | 激进压缩 |
| INT2 | 2 bit | 研究型 |

## 2. 按量化对象分类

### 2.1 权重量化

只压缩模型参数，常见写法：

- `W4A16`
- `W8A16`

优点：

- 路径稳定
- 工程上更容易落地

### 2.2 权重 + 激活量化

同时量化 weight 和 activation，常见写法：

- `W8A8`
- `W4A8`

优点：

- 理论上更能降低带宽和算力压力

缺点：

- activation 更难处理
- 更依赖校准和 kernel 设计

### 2.3 KV Cache 量化

随着上下文变长，KV cache 会占大量显存，所以也常单独量化。

常见写法：

- `KV8`
- `KV4`

## 3. 端侧常见精度组合

在端侧部署里，常见组合主要有：

- `W4A16`
- `W8A8`
- `W8A16`
- `W4A16 + KV8`

## 4. 一条工程经验

不要把“bit 更低”直接等同于“推理更快”。

真正决定效果的通常是：

- kernel 能否高效跑起来
- 是否减少了总访存
- unpack / dequant 的成本是否值得

## 5. 面试速记版

- 量化精度最直观的分类就是 bit 数
- LLM 推理里最常见的是 `FP16`、`INT8`、`INT4`
- 工程表达里常用 `W4A16`、`W8A8` 这种写法
- 低 bit 不一定更快，但通常更省显存
