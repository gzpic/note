# 计算图的核心要素

## 1. 最精简的三个要素

如果从推理框架 / ONNX / 深度学习计算图的角度看，一个计算图最核心可以抽象成 3 个要素：

- Node
- Tensor
- Attribute

也可以说：

> Graph = Node + Tensor + Attribute

## 2. Node：节点

节点表示计算操作，也就是算子。

常见例子：

- `MatMul`
- `Conv`
- `Add`
- `Relu`
- `Softmax`
- `LayerNorm`

节点回答的是：

> 这里要做什么计算。

## 3. Tensor：边 / 数据

边本质上是数据流，在深度学习图里通常就是 Tensor。

例如：

- 输入张量
- 中间结果
- 输出张量
- 权重张量

数据流可以理解成：

```text
Tensor → Node → Tensor
```

## 4. Attribute：节点参数

属性是算子的配置参数，例如：

- `Conv` 的 `kernel_size / stride / padding`
- `Softmax` 的 `axis`
- `LayerNorm` 的 `epsilon`

属性回答的是：

> 这个算子要按什么规则执行。

## 5. 工程里的一个常见扩展

在真实推理框架里，除了这 3 个要素，通常还会非常关注：

- shape
- layout

因为很多优化其实都发生在：

- shape 推导
- layout 变换
- 子图切分
- fuse pass

## 6. 面试速记版

- 计算图最核心的三要素是 Node、Tensor、Attribute
- Node 表示算子
- Tensor 表示数据流
- Attribute 表示算子参数
