# 属性存在哪里

## 1. 结论

在标准计算图，尤其是 ONNX 里，属性通常存储在节点上。

也就是：

> Attribute 属于 Node，而不是属于 Edge。

## 2. 为什么属性属于节点

因为属性描述的是“算子怎么执行”，不是“数据是什么”。

例如：

- `Conv` 的 `kernel_size`
- `stride`
- `padding`
- `Softmax` 的 `axis`
- `LayerNorm` 的 `epsilon`

这些都属于算子的行为配置。

## 3. 为什么不放在边上

边主要表示 Tensor，本质上回答的是：

- 数据形状是什么
- 数据类型是什么
- 数据内容是什么

而属性是算子级别的执行参数，不是张量本身的属性。

## 4. 一个简单例子

例如 `Conv` 节点：

- 输入 Tensor：`X`
- 权重 Tensor：`W`
- 输出 Tensor：`Y`
- 属性：`kernel_size / stride / padding`

这里很明显：

- `X / W / Y` 是边
- `kernel_size / stride / padding` 是节点参数

## 5. 工程上的补充

虽然在标准 ONNX 里属性属于节点，但在一些更底层的 IR 或优化系统里，某些属性也可能被张量化或常量化，以便统一做优化。

所以：

- 在标准抽象里，Attribute 在 Node 上
- 在更底层实现里，可能会被进一步转换

## 6. 面试速记版

- 属性通常是节点参数
- Tensor 是边
- 权重属于常量 Tensor
- 属性不属于边，而是属于节点
