## 1️⃣ 访存相关优化（Memory-bound 的核心）

### 1.1 Memory coalescing（最基础，但最重要）

**目标**：

- 一个 warp → 连续、对齐访问 global memory

典型手段：

- `float4 / half2 / int4` 向量化 load/store
- layout 调整（NCHW → NHWC）
- 避免 stride 访问

👉 **没 coalescing，后面优化基本白搭**

---

### 1.2 Shared memory tiling

**你 register tiling 的前一层**

```plain text
Global →Shared → Register → Compute


```

- block 级复用
- 减少 global 带宽压力

常见于：

- GEMM
- Attention QK
- Conv

---

### 1.3 Bank conflict 消除

- padding shared memory
- 调整 layout（例如 `[tile][k+1]`）

👉 很多“理论上没问题但跑得慢”的 kernel 就死在这

---

### 1.4 Read-only cache / texture cache

- `__ldg()`
- const memory
- texture fetch（对随机访问友好）

典型：

- embedding
- lookup table
- position encoding

---

## 2️⃣ 计算相关优化（Compute-bound）

### 2.1 Register tiling（你已点名）

**核心思想**：

> 把一个线程的“算术密度”拉高

代价：

- register 数上升
- occupancy 下降

👉 本质 trade-off：

**occupancy vs ILP（Instruction Level Parallelism）**

---

### 2.2 指令级并行（ILP）

- unroll loop
- 手工重排指令
- 避免 RAW dependency

```c++
// bad
a =load();
b = a * x;

// better
a0 =load();
a1 =load();
b0 = a0 * x;
b1 = a1 * x;


```

---

### 2.3 使用专用指令

- `HFMA2 / HMMA`
- Tensor Core（wmma / mma）
- dp4a / dp2a（int8）

👉 **这是“换挡提速”级别的优化**

---

## 3️⃣ 并行结构优化（Occupancy & Scheduling）

### 3.1 Occupancy 调优

影响因素：

- registers / thread
- shared memory / block
- block size

不是越高越好，而是：

> “足够隐藏 latency 即可”

很多高性能 kernel 的 occupancy 只有 25%～50%

---

### 3.2 Warp-level programming

- `__shfl_sync`
- warp reduce / broadcast
- warp specialization

避免：

- shared memory
- `__syncthreads()`

👉 Attention / reduction 类算子非常吃这个

---

### 3.3 Warp specialization（高级技巧）

不同 warp 干不同事：

- warp0：load
- warp1：compute
- warp2：store

TensorRT、FlashAttention 都用得很多

---

## 4️⃣ Pipeline / 重叠优化（你提到的重点）

### 4.1 Double buffer（你已点名）

```plain text
load tile k+1  ┐
compute tile k ├→ overlap


```

- shared double buffer
- register double buffer

---

### 4.2 软件流水（software pipeline）

把 loop 拆成：

- prefetch
- compute
- writeback

配合：

- `#pragma unroll`
- 手工调度

---

### 4.3 Async copy（Ampere+）

```c++
cp.async.shared.global


```

- global → shared 异步
- 减少 `__syncthreads()` 开销

👉 FlashAttention / CUTLASS 的核心之一

---

## 5️⃣ Kernel 级 & 系统级优化（容易被忽略）

### 5.1 Kernel fusion

- 消除中间 tensor
- 减少 launch + memory traffic

例子：

- bias + activation
- rope + matmul
- softmax fuse

---

### 5.2 减少 kernel launch overhead

- CUDA Graph
- persistent kernel
- 合并小 kernel

👉 **对 decode 阶段尤其重要**（你之前也问过）

---

### 5.3 Layout / 算法级重构

- 改 attention 算法（Flash / Paged / Blocked）
- 改 reduction 顺序
- 改 tile 形状

👉 **这是“算法级 > 指令级”的优化**
