# C++ 100题

## 1. 智能指针与 RAII

从 `[[CPP&C/知识点整理/3.智能指针和raii]]` 提炼出的核心点：

- 野指针：未初始化或非法来源的指针。
- 悬空指针：曾经指向有效对象，但对象已经释放。
- 空指针：值为 `nullptr`，不指向有效对象。
- 避免野指针/悬空指针的方法：
  - 初始化置空
  - 申请后判空
  - 释放后置空
  - 使用智能指针
- `unique_ptr`：
  - 独占所有权
  - 只能移动，不能复制
  - 原始裸指针不要再被其他智能指针重复接管
- `shared_ptr`：
  - 通过控制块维护引用计数
  - 只保证计数修改线程安全，不保证对象本身线程安全
  - 赋值的关键是旧资源安全释放和自赋值安全
- `weak_ptr`：
  - 不增加引用计数
  - 用来打破循环引用

## 2. 内存管理

从 `[[CPP&C/知识点整理/4.内存管理]]` 提炼出的核心点：

- 内存分区：
  - 栈区
  - 堆区
  - 全局/静态区
  - 常量区
  - 代码区
- 堆和栈的区别：
  - 堆大栈小
  - 栈自动释放，堆手动释放
  - 堆易产生外部碎片，栈基本不会
- `new/delete` 与 `malloc/free`：
  - `new/delete` 管对象生命周期
  - `malloc/free` 只管原始内存
- `delete[]` 的本质：
  - 数组对象需要记录长度
  - 需要逐个析构，再整体释放
- 深拷贝和浅拷贝：
  - 浅拷贝只复制指针值
  - 深拷贝复制底层资源
- 指针和引用区别：
  - 指针可变向、可为空、可多级
  - 引用必须初始化，本质是别名
- 内存对齐：
  - 成员偏移按自身对齐
  - 结构体总大小按最宽基本类型对齐
  - 合理排列成员可减少内部碎片
- 常见内存错误：
  - 未分配就使用
  - 未初始化就使用
  - 越界
  - 泄漏
  - 释放后继续使用

## 3. 关键字

从 `[[CPP&C/知识点整理/5.关键字]]` 提炼出的核心点：

- `const`
  - 修饰对象只读
  - `const` 对象只能调 `const` 成员函数
- `static`
  - 函数内静态局部变量只初始化一次
  - 类静态成员属于类，不属于对象
  - 文件作用域 `static` 用于隐藏符号
- `extern`
  - 声明定义在其他编译单元
- `extern "C"`
  - 关闭 C++ 名字修饰，兼容 C 接口
- `explicit`
  - 禁止隐式构造/隐式转换
- `constexpr`
  - 编译期常量或编译期可求值函数
- `volatile`
  - 告诉编译器不要把读写优化成寄存器缓存，常见于寄存器映射和并发可见性场景
- `mutable`
  - 允许在 `const` 成员函数中修改某些成员

## 4. STL 与容器

从 `[[CPP&C/知识点整理/6.STL]]` 提炼出的核心点：

- `vector`
  - 底层是动态数组
  - 尾插摊还复杂度通常是 `O(1)`
  - `push_back` 与 `emplace_back` 语义不同
  - 删除元素常见 `erase` 与 `erase-remove`
- `list`
  - 双向链表
- `deque`
  - 分段连续空间
  - 首尾插删高效，随机访问常数大于 `vector`
- `stack/queue`
  - 容器适配器，默认常基于 `deque`
- `priority_queue`
  - 堆结构，常用 `vector` 实现
- `map/set`
  - 红黑树
  - 面试常问红黑树性质和复杂度
- `unordered_map/unordered_set`
  - 哈希表
  - 平均 `O(1)`，最坏退化

## 8. 追加高频题

### 8.1 智能指针

- `unique_ptr`
  - 独占所有权
  - 不能拷贝，只能移动
- `shared_ptr`
  - 共享所有权
  - 内部有引用计数
- `weak_ptr`
  - 不增加引用计数
  - 常用来打破循环引用

补充：

- 循环引用会导致 `shared_ptr` 引用计数始终不归零，进而内存泄漏。
- `weak_ptr` 更像观察者，不拥有对象。

### 8.2 `new` 和 `malloc`

- `new`
  - 分配内存并调用构造函数
- `malloc`
  - 只分配原始内存，不调用构造函数
- `delete`
  - 调用析构再释放内存
- `free`
  - 只释放内存

### 8.3 `vector` 的底层存储

- `vector` 底层是连续内存。
- 动态扩容时通常会申请更大连续空间，再搬运旧元素。
- `new vector<T>` 只是把 `vector` 对象本身放在堆上，不改变它内部连续存储的特性。

### 8.4 `push_back` 和 `emplace_back`

- `push_back`
  - 传入一个已有对象
  - 再拷贝或移动进容器
- `emplace_back`
  - 传构造参数
  - 直接在容器尾部原地构造

### 8.5 C++ 继承与多态底层

- 多态一般依赖虚函数表。
- 含虚函数的对象通常会带一个虚表指针。
- 调用虚函数时会通过虚表做动态绑定。
