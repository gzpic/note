# 推理步骤

## 1. 端到端推理主流程

综合 `[[ARM/大模型基础/大模型推理核心/公式推导]]`、`[[CUDA/自研算子库/执行步骤新/执行步骤新]]` 提炼：

1. 输入 token
2. Embedding
3. 每层执行：
   - RMSNorm / LayerNorm
   - QKV GEMM
   - 写入 KV Cache
   - Attention
   - Output GEMM
   - Residual
   - FFN
   - Residual
4. Final Norm
5. LM Head GEMM
6. 得到 logits
7. 采样得到下一个 token

## 2. Prefill

- 一次处理整段 prompt
- attention 更像大矩阵计算
- 核心瓶颈通常是 GEMM 和大规模计算吞吐

## 3. Decode

- 一次只推进少量 token，常见是 1 token
- 当前 token 产生 Q
- 历史 token 的 K/V 从 cache 读取
- 核心瓶颈通常是 KV cache 访问和带宽

## 4. logits

从 `[[ARM/大模型基础/大模型推理核心/公式推导]]` 提炼：

- logits 是词表上每个 token 的未归一化分数
- decode 阶段常见 shape：
  - `[batch_size, vocab_size]`
- prefill 阶段常见输出：
  - `[T, vocab_size]`
  - 但通常只取最后位置用于生成下一个 token

## 5. 这一页可直接背的差异总结

- Prefill 更偏计算密集
- Decode 更偏访存密集
- 同一套层结构在两个阶段都存在，但 attention 和 KV cache 读写方式完全不同

## 6. Self-Attention 和 Cross-Attention

- Self-Attention
  - `Q/K/V` 都来自同一个序列
  - 本质是建模序列内部元素关系
- Cross-Attention
  - `Q` 来自当前序列
  - `K/V` 来自另一个序列
  - 本质是建模两个序列之间的对应关系

面试简答：

- self 看自己内部依赖，cross 看两个序列之间的信息对齐。

## 7. 为什么 Attention 要除以 `sqrt(dk)`

- 点积维度越大，数值幅度通常越大。
- 如果不缩放，softmax 很容易进入特别陡的区域。
- 这样会导致梯度过小或者训练不稳定。

面试简答：

- 除以 `sqrt(dk)` 是为了控制点积尺度，让 softmax 更稳定。

## 8. 为什么 Transformer 里需要 FFN

- Attention 负责让不同 token 之间交换信息。
- FFN 负责对每个 token 做更强的非线性变换。
- 它相当于在注意力之后给每个 token 一个“加工厂”。

## 9. Attention Mask

- mask 一般加在 `QK^T` 之后、softmax 之前。
- 常见有两类：
  - Padding Mask
  - Causal Mask
- 实现上通常是把不合法位置加一个很小的负数。

## 10. KV Cache 为什么没有 Q Cache

- Q 只和当前步有关，用完就没价值。
- K/V 会在后续很多步重复使用。
- 所以 decode 阶段缓存 K/V 可以显著减少重复计算，而缓存 Q 没什么收益。

## 11. FlashAttention 的算法流程

- 把 `Q/K/V` 切成多个 tile
- 每次加载一块 `K/V`
- 与当前 `Q` 块做乘法
- 用 online softmax 维护 running max 和 running denominator
- 计算完这一块的贡献后直接累加到输出
- 中间大矩阵不完整落显存

面试简答：

- FlashAttention 的核心不是换公式，而是重排计算顺序，让计算驱动访存，减少中间结果写回显存。

## 12. Prefix Cache 和投机解码

### 12.1 Prefix Cache

- 多个请求如果共享 prompt 前缀，可以直接复用前缀对应的 KV Cache。
- 这样可以降低重复 prefill 成本。

### 12.2 投机解码

- 用一个小模型先猜若干 token
- 再用大模型并行验证
- 猜对就一次接收多个 token
- 猜错就回退到正确位置继续生成

面试简答：

- 投机解码的本质是用更便宜的模型提前生成候选，再用大模型批量验证，换取更高吞吐和更低平均延迟。

## 13. Dropout、BatchNorm、LayerNorm

### 13.1 Dropout

- 训练时随机屏蔽一部分神经元
- 推理时不再屏蔽，使用完整网络

### 13.2 BatchNorm 和 LayerNorm

- BatchNorm
  - 统计 batch 维度
  - 更常见于 CNN
- LayerNorm
  - 统计单样本内部特征维度
  - 更适合 Transformer 和大模型

面试简答：

- 大模型里 LayerNorm 明显更常见，因为它不依赖 batch 统计，更适合变长序列和小 batch 场景。

## 14. GQA 再补一版直观解释

- 假设 Q 有很多头，但 K/V 头更少。
- 每组 Q 头共享同一组 K/V 头。
- 这样做会让 KV Cache 更小，decode 阶段读写压力更低。

## 15. Prefix / Prefill / Decode 一起怎么讲

- Prefix Cache 是跨请求复用已有前缀 KV。
- Prefill 是对当前请求的新 prompt 做整段计算。
- Decode 是在已有 KV 基础上逐 token 生成。

面试里串起来说：

- 如果请求之间有公共前缀，先用 prefix cache 复用；不能复用的部分再做 prefill；之后进入 decode 按 token 生成。
