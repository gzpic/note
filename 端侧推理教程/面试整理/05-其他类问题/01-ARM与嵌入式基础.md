# ARM 与嵌入式基础

## 1. ARM 架构过渡

从 `[[ARM/ARM]]` 提炼：

- 从 M4 到 A 系列要先建立思维切换：
  - M 系列更强调实时性和确定性
  - A 系列更强调高性能和复杂系统能力
- 核心对比维度：
  - 流水线
  - 顺序执行 vs 乱序执行
  - MPU vs MMU
  - cache 系统

## 2. 为什么 M4 一般不用 cache

从 `[[ARM/ARM]]` 提炼：

- 目标是确定性
- cache miss 会引入不可预测延迟
- 更常见做法是直接访问 SRAM/Flash，或使用 TCM

## 3. C 语言与寄存器/栈

从 `[[嵌入式基础/c语言基]]` 提炼：

- ARM 调用约定里：
  - `R0-R3` 常作参数和返回值
  - `R4-R11` 常作局部变量保存
  - `SP` 指向当前栈顶
- `MSP` 和 `PSP` 的差别
- `NVIC`、`SysTick`、`SCB` 等系统寄存器

## 4. RTOS 调度

从 `[[嵌入式基础/rtos/rtos]]`、`[[嵌入式基础/rtos/tick]]`、`[[嵌入式基础/rtos/rtos链表]]`、`[[嵌入式基础/rtos/调度策略]]` 提炼：

- FreeRTOS 核心对象是 TCB
- `pxTopOfStack` 是当前上下文切换最关键成员之一
- 就绪队列按优先级拆分
- `vTaskSwitchContext()` 负责挑选下一个任务
- 同优先级任务可时间片轮转
- 高优先级任务就绪后可抢占当前任务

## 5. 芯片基础

从 `[[嵌入式基础/芯片基础知识]]` 提炼：

- 流水线越深，分支预测失败代价越大
- 冒险处理会增加控制复杂度
- 动态调度的本质是选择当前可执行且不破坏依赖的指令
