# C++ 与现代 C++

## 高频题

### 1. RAII 和智能指针

- `RAII` 的核心是“资源绑定对象生命周期”
- 构造时拿资源，析构时放资源
- 面试里通常要顺手带出：
  - 为什么减少泄漏
  - 为什么异常安全
  - 为什么比手写 `new/delete` 稳

### 2. `unique_ptr`、`shared_ptr`、`weak_ptr`

- `unique_ptr`
  - 独占所有权
  - 不能拷贝，只能移动
- `shared_ptr`
  - 共享所有权
  - 通过控制块维护引用计数
  - 引用计数线程安全，不代表对象本身线程安全
- `weak_ptr`
  - 不增加引用计数
  - 主要用于打破循环引用

### 3. `new/delete` 和 `malloc/free`

- `new/delete` 管对象生命周期
- `malloc/free` 只管原始内存
- `new` 可能抛异常，`malloc` 返回空指针
- 面试稳妥回答：
  - C++ 对象优先 `new/delete` 或智能指针
  - C 风格裸内存才考虑 `malloc/free`

### 4. 多态和虚函数

- 多态常见实现依赖虚函数表
- 含虚函数的对象通常带虚表指针
- 运行时通过虚表做动态绑定
- 常见追问：
  - 构造和析构时虚函数行为
  - 纯虚函数和抽象类
  - 为什么基类析构常写成虚析构

### 5. `vector` 为什么快，什么时候会慢

- 底层是连续内存
- 尾插通常摊还 `O(1)`
- 随机访问快，缓存友好
- 但中间插入删除代价高
- 扩容会触发重新分配和搬运

### 6. `push_back` 和 `emplace_back`

- `push_back` 放已有对象
- `emplace_back` 传构造参数，原地构造
- 不是所有场景 `emplace_back` 都更快
- 面试别把它答成“永远比 `push_back` 高级”

### 7. 左值、右值、移动语义

- 右值引用主要是为了转移资源而不是复制资源
- `std::move` 本质是显式转成右值
- 移动不是“内存不变”，而是“资源所有权转移”
- 常见追问：
  - 移动后对象还能不能用
  - 返回值优化和移动构造是什么关系

### 8. `const`、`static`、`extern`、`volatile`

- `const`
  - 表示只读语义
- `static`
  - 可用于静态存储期、类静态成员、文件内可见性
- `extern`
  - 声明定义在别处
- `volatile`
  - 告诉编译器不要把读写简单缓存优化
  - 常见于寄存器映射和特殊并发可见性场景

### 9. 模板和泛型

- 模板本质是编译期代码生成
- 高频点：
  - 函数模板和类模板
  - 模板特化
  - 隐式实例化和显式实例化
- 面试容易追问：
  - 模板和宏的区别
  - 为什么模板报错复杂

### 10. 内存对齐

- 成员偏移要满足自身对齐要求
- 结构体总大小通常按最大对齐单位补齐
- 合理调整成员顺序可减少内存浪费
- 工程上会直接影响：
  - 空间占用
  - cache 友好性
  - SIMD 访问

## 回答骨架

### C++ 基础题的稳妥答法

1. 先给定义
2. 再讲底层机制
3. 再讲工程意义
4. 最后补一个常见坑

例如“`shared_ptr` 是什么”可以直接答：

- `shared_ptr` 是共享所有权智能指针，底层通过控制块维护引用计数。
- 它适合多个对象共同持有同一资源，但只保证引用计数操作线程安全，不保证对象本身线程安全。
- 工程里最常见的坑是循环引用，所以常配合 `weak_ptr` 使用。

## 易错点

- 不要把“引用计数线程安全”等同于“对象线程安全”
- 不要把 `emplace_back` 说成所有情况都更优
- 不要把移动语义说成“单纯地址拷贝”
- 不要把 `volatile` 说成“线程同步方案”
- 不要把多态回答成只有“虚函数”这一个点，要能补到虚表和动态绑定
