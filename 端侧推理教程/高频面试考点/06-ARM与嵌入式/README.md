# ARM 与嵌入式

## 高频题

### 1. Cortex-M 和 Cortex-A 区别

- `M` 系列
  - 更偏实时控制
  - 更强调确定性
  - 常见于 MCU
- `A` 系列
  - 更偏高性能和复杂系统
  - 常跑 Linux
  - 更强调 cache、MMU、乱序执行

### 2. 为什么很多 MCU 场景不强调 cache

- 因为 cache miss 会带来不可预测延迟
- 实时场景更看重确定性
- 常见替代思路：
  - SRAM
  - TCM

### 3. MPU 和 MMU 区别

- `MPU`
  - 更偏内存保护
  - 不做完整虚拟内存映射
- `MMU`
  - 支持地址转换、分页、虚拟内存
  - 更常见于 A 系列和通用 OS

### 4. 中断和上下文切换

- 中断是外设或系统事件打断当前执行流
- 上下文切换本质是保存当前现场，恢复目标任务现场
- 高频追问：
  - 为什么中断要尽量短
  - 为什么中断里不适合做重逻辑

### 5. FreeRTOS 调度

- 核心对象是 `TCB`
- 常见是优先级调度
- 同优先级任务可时间片轮转
- 高优先级任务就绪后可抢占低优先级任务

### 6. `tick` 的作用

- 提供系统节拍
- 让 RTOS 知道时间推进
- 常用于：
  - 延时
  - 超时
  - 时间片切换

### 7. DMA 在嵌入式里怎么讲

- DMA 让数据搬运交给专门控制器
- 降低 CPU 搬运负担
- 典型场景：
  - 串口
  - SPI
  - 摄像头
  - 音频

### 8. 寄存器、栈和调用约定

- ARM 调用里常见：
  - `R0-R3` 传参和返回值
  - `SP` 指向栈顶
- 这类题常和：
  - 函数调用
  - 中断现场保护
  - 上下文切换
 绑在一起问

## 回答骨架

### 嵌入式题的稳妥答法

1. 先说目标约束是什么
2. 再说硬件或系统机制
3. 再说为什么这样设计
4. 最后补工程代价

例如“为什么 M4 一般不强调 cache”：

- 因为它主要面向实时控制，更关注响应确定性。
- cache hit 很快，但 miss 会引入不可预测延迟。
- 所以很多 MCU 更愿意用 SRAM、TCM 或简单存储体系来换可预测性。

## 高频口头结论

### RTOS 和 Linux 的核心差异怎么讲

- RTOS 更看重实时性和确定性
- Linux 更看重通用性、吞吐和复杂资源管理

### 为什么端侧 ARM 平台经常要同时懂系统和硬件

- 因为很多问题不是单纯算法问题
- 往往同时牵涉：
  - cache
  - 带宽
  - DMA
  - 中断
  - 调度

## 易错点

- 不要把 MPU 和 MMU 说成一回事
- 不要把 DMA 说成“完全不需要 CPU”
- 不要把 RTOS 只理解成“任务切换器”，它还涉及时钟、中断、同步对象和内存管理
