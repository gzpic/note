# 系统基础与问题定位速记

## 体系结构与并发
- 线程/进程/协程：进程资源隔离最强，线程共享地址空间，协程切换最轻量。
- DMA：外设可直接与内存搬运数据，降低 CPU 参与度。
- cache 基础：利用时间/空间局部性，命中率直接影响性能与时延抖动。

## 内存与执行问题
- 内存分配步骤：虚拟地址申请 -> 映射建立 -> 缺页分配 -> 回收复用。
- 为什么 pinned memory 快：减少分页中转，DMA 直达更高效。
- H2D 与 D2H 能重叠吗：可以，前提是异步拷贝、不同 stream、硬件支持并发引擎。

## 定位方法
- 时延长怎么定位：拆分 TTFT/prefill/decode，先看时间线再下钻热点 kernel。
- 抖动怎么定位：查调度波动、缓存命中、频率降档、内存回收与并发突发。
- 常用流程：先现象分层 -> 指标定位 -> 复现实验 -> 最小改动验证。
- 工具组合：`perf` 看 CPU、`nsys` 看链路、`ncu` 看 kernel 微观瓶颈。

## CUDA/推理实战追问
- `cuda_device_max_connection` 含义：影响并发连接与调度并行度，需结合 workload 调参。
- stream 使用前提：异步 API + 依赖关系清晰 + 必要同步点正确。
- block 内通信方式：shared memory + barrier，或 warp shuffle。
- compute/memory 之外还有什么 bound：launch-bound、latency-bound、sync-bound、IO-bound。

## 面试答题模板
- 先给结论：是算力、带宽、调度还是数据问题。
- 再给证据：指标/火焰图/时间线/热点函数。
- 最后给动作：改什么参数、改哪个算子、如何验证收益和回归风险。
