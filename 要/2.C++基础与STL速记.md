# C++基础与STL速记

## 面向对象
- 继承/封装/多态：继承是复用，封装是隐藏实现，多态是同接口不同行为。
- 编译期多态/运行期多态：编译期靠模板与重载，运行期靠虚函数。
- 虚函数/纯虚函数：虚函数可重写，纯虚函数必须重写并使类变抽象类。
- 抽象类：含纯虚函数的类，不能实例化，只能作为基类接口。
- 虚指针/虚表：对象里有 `vptr` 指向 `vtable`，运行期按表查函数地址。

## 指针与内存
- 智能指针：`unique_ptr` 独占，`shared_ptr` 共享，`weak_ptr` 打破循环引用。
- 智能指针和普通指针区别：智能指针自动管理生命周期，普通指针需要手动释放。
- RAII：资源绑定对象生命周期，构造拿资源，析构释放资源。
- 野指针/悬空指针：野指针未初始化；悬空指针指向已释放内存。避免方式是初始化、置空、少用裸指针。
- 空指针：值为 `nullptr`，表示不指向有效对象。
- 内存泄漏定义：申请后未释放导致资源丢失；避免靠 RAII，排查靠 ASan/Valgrind/泄漏检测。
- `new/delete` 与 `malloc/free`：前者会调构造析构，后者只分配原始内存。
- `new[]/delete[]`：数组必须配对，`delete[]` 会逐元素析构。
- 内存对齐：按对齐边界放置数据，减少访存次数，提高性能。
- 深拷贝/浅拷贝：浅拷贝复制指针地址，深拷贝复制指针指向内容。
- 内存分配步骤：申请虚拟地址 -> 建映射 -> 首次访问触发缺页 -> 分配物理页。
- 栈和堆区别：栈自动管理速度快容量小；堆手动管理灵活但易碎片与泄漏。
- 指针和引用区别：引用必须绑定对象且不可改绑；指针可空、可改指向。
- `const *` 组合：`const int*` 不能改值；`int* const` 不能改指针；`const int* const` 都不能改。

## 语法与编译
- `std::move`：把左值转右值引用，允许触发移动语义。
- 完美转发：`std::forward<T>(x)` 保留值类别，常配合模板参数转发。
- 几种转型：`static_cast` 常规转换；`dynamic_cast` 运行期安全下转；`const_cast` 去/加 const；`reinterpret_cast` 低层重解释。
- 编译过程：预处理 -> 编译 -> 汇编 -> 链接。
- 静态链接/动态链接：静态链接体积大但独立；动态链接可复用库但有运行依赖。
- 编译优化 `-O0/-O1/-O2/-O3`：优化级别越高通常越快，但编译更慢且可能影响调试与代码体积。
- `constexpr`：可在编译期求值的常量/函数，减少运行期开销。

## STL容器与线程安全
- `vector` 原理：连续内存，扩容时搬迁，随机访问 O(1)。
- `map` 原理：红黑树，查找/插入/删除 O(logN)，有序。
- `unordered_map` 原理：哈希表，平均 O(1)，最坏 O(N)。
- `map` 和 `unordered_map` 区别：前者有序且稳定复杂度，后者无序但平均更快。
- `array`：定长连续内存，栈上常见，零额外开销。
- `list`：双向链表，插删快，随机访问慢。
- `set`：通常红黑树实现，自动去重且有序。
- STL线程安全吗：只读通常可并发；写或读写并发一般不安全，需要外部同步。

## 算法与手撕
- 常见排序：冒泡/插入/选择 O(N^2)；快排平均 O(NlogN)；归并稳定 O(NlogN)；堆排 O(NlogN)。
- 排序使用场景：小数据可插入；需要稳定可归并；平均性能优先多用快排。
- `memcpy` 实现思路：按字节/字宽拷贝，考虑对齐；重叠内存应使用 `memmove`。
- `vector` 手撕思路：三指针管理 `begin/end/cap`，支持扩容、移动构造、析构释放。
- 单例模式两种：饿汉式（启动即建）；懒汉式（首次用才建，C++11 局部静态线程安全）。

## 设计模式
- 工厂模式：把对象创建与使用解耦，便于扩展不同产品。
- 责任链模式：请求沿链传递，直到某节点处理，降低耦合。
- 单例模式：全局唯一实例，适合配置与资源管理（慎用全局状态）。
