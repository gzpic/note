# 模型部署量化与推理速记

## Transformer与结构
- Transformer vs RNN：Transformer 并行强、长依赖建模好；RNN 时序强但并行差。
- 为什么 SwiGLU：门控提升表达能力，通常比 ReLU/GELU 在同算力下效果更好。
- Decoder-only 为什么多：生成任务统一、训练和推理链路简单、生态成熟。
- RoPE 是什么：旋转位置编码，把位置信息注入 Q/K，长上下文外推更友好。

## 模型转换与部署
- 模型转换：PyTorch/TF -> ONNX/TRT/其他后端图。
- 导出不支持算子：自定义插件、图替换、前后处理下沉、算子拆解。
- 精度掉点排查：先对齐预处理/后处理，再逐层比对中间激活与算子实现。
- 返回值不对：先查输入归一化、tokenizer、dtype、layout、mask 与后处理。
- 性能不达标：先分 prefill/decode，查热点算子、带宽、并发、缓存命中。
- 模型优化常见项：冗余消除、常量折叠、算子融合、内存规划、调度优化。

## 常用指标
- 时间：TTFT、decode 单 token 时延、端到端时延。
- 内存：峰值内存、稳态内存、KV cache 占用。
- 带宽：峰值带宽、有效带宽利用率。
- 功耗：平均功耗与温升。
- 稳定性：长时间运行是否抖动、是否 OOM、是否降频。

## 量化核心问题
- 为什么常用对称量化：实现简单、硬件友好、整数路径高效，尤其权重量化常见。
- 非对称何时用：激活分布偏移明显时更合适，但实现复杂度更高。
- GPTQ vs AWQ：GPTQ 偏后验误差最小化；AWQ 偏保护关键权重通道，实战常看模型和硬件适配。
- 激活值难量化：动态范围大、分布随输入变化，容易引起饱和和精度损失。
- 哪些参数能量化：线性层权重最常见；激活、KV、部分归一化层要谨慎。
- 量化步骤：校准数据 -> 统计分布 -> 选粒度/bit -> 量化/反量化插入 -> 验证精度与性能。
- 极不均匀分布（如最小值 -10000）如何量化：做 clipping、分段/对数缩放、离群值单独处理或混合精度保留。

## 推理阶段优化
- prefill 技术：大 batch、算子融合、FlashAttention、图优化。
- decode 技术：KV cache 布局、paged/cache 复用、小 kernel 合并、低 launch 开销。
- 什么时候优先调度优化：当 GPU 利用率不高且有明显空泡/等待时。
