# 操作系统与 Linux

## 高频题

### 1. 进程和线程区别

- 进程是资源分配基本单位
- 线程是调度基本单位
- 同一进程内线程共享地址空间
- 常见延伸：
  - 为什么线程切换通常更轻
  - 为什么多线程同步更复杂

### 2. 进程通信和线程同步

- 进程通信常见：
  - 管道
  - 消息队列
  - 共享内存
  - socket
- 线程同步常见：
  - 互斥锁
  - 条件变量
  - 读写锁
  - 信号量
  - 自旋锁

### 3. 互斥锁、自旋锁、信号量

- 互斥锁
  - 竞争不到通常睡眠
- 自旋锁
  - 竞争不到忙等
  - 适合持锁时间极短
- 信号量
  - 本质是计数器
  - 可做互斥，也可做同步

### 4. 虚拟内存、页表、TLB

- 虚拟内存给进程提供独立且更连续的地址空间抽象
- 页表负责虚拟页到物理页框映射
- `TLB` 是页表项的高速缓存
- 高频追问：
  - 缺页中断是什么
  - 为什么需要 TLB

### 5. `mmap`、DMA、零拷贝

- `mmap`
  - 把文件或匿名内存映射到进程虚拟地址空间
- `DMA`
  - 解决“谁来搬数据”
- 零拷贝
  - 解决“能不能少搬几次”

一组最稳的区分：

- DMA 是搬运机制
- mmap 是映射方式
- 零拷贝是优化目标

### 6. `select`、`poll`、`epoll`

- `select`
  - 位图
  - 每次都要拷贝和扫描
- `poll`
  - 数组
  - 仍然线性扫描
- `epoll`
  - 注册和就绪分离
  - 更适合大量连接、少量活跃

### 7. page fault 和 OOM

- `OOM`
  - 放不下
- `page fault`
  - 页访问时需要操作系统处理
- 在大模型端侧场景里：
  - OOM 常来自权重、KV Cache、workspace
  - page fault 常来自 `mmap`、按需分配、UMA 迁移

### 8. Linux 常见命令怎么讲

- 看进程：`ps`、`top`
- 看内存：`free`
- 看系统状态：`vmstat`
- 看热点：`perf`
- 面试里重点不是背命令，而是知道它们的分工

## 回答骨架

### 操作系统题最稳的回答结构

1. 先给定义
2. 再说为什么设计成这样
3. 再说代价和适用场景
4. 最后补一条工程联系

例如“为什么要虚拟内存”：

- 它的核心价值是隔离、扩展和便于管理。
- 通过页表和 MMU，进程看到的是独立虚拟地址空间，而不是直接操作物理内存。
- 代价是地址转换开销，所以需要 TLB 加速。
- 工程上 page fault、swap 和内存局部性都会直接影响程序稳定性和延迟。

## 高频口头题

### 多进程和多线程怎么选

- 共享数据多、协作紧：偏多线程
- 隔离性、稳定性要求高：偏多进程
- 实际工程里常是混合使用

### 为什么线程仍然要加锁，哪怕是单核

- 单核也会发生抢占式调度
- 只要存在共享数据的并发访问，就仍然需要同步保护

### `free` 看的是什么

- 系统级物理内存和 swap
- 不是单个进程的虚拟地址空间
- 判断内存是否真的紧张，更该看 `available`

## 易错点

- 不要把线程说成“资源分配单位”
- 不要把 `free` 当成查看虚拟内存空间的工具
- 不要把 `mmap` 和 DMA 说成一回事
- 不要说 `epoll` 永远更快，正确说法是更适合高并发活跃连接较少的场景
